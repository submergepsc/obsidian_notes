直接上干货。掌握 Obsidian 的原生增强语法比装一堆花哨的插件更靠谱。这些原生语法在你的笔记库跨多端设备同步时绝对稳定，不会出现渲染兼容问题。

以下是最全面、最常用的 Obsidian 专属与进阶 Markdown 语法速查表：

1. Obsidian 核心：双向链接与块引用 (WikiLinks)

这是 Obsidian 的灵魂所在，用于建立知识网络。

功能 语法 说明
基础双链 [[页面名称]] 链接到库中的另一个文件。
别名显示 [[页面名称\|显示的文本]] 链接到指定页面，但在当前文中显示为“显示的文本”。
锚点：链接到标题 [[页面名称#某个小标题]] 精准跳转到目标文档的具体标题位置。
锚点：链接到区块 [[页面名称^区块ID]] 链接到目标文档的某一段落或列表项（输入 ^ 后 Obsidian 会自动弹出区块列表供你搜索选择）。
嵌入/透视 (Transclusion) ![[页面、图片或PDF]] 在当前光标处直接渲染并显示目标文件的内容。如果是笔记，会嵌入全文；如果是 ![[页面#标题]]，则只嵌入该标题下的内容。

2. 进阶排版与高亮

标准 Markdown 之外的扩展标记，非常适合做读书笔记和知识提取。

功能 语法 说明
文字高亮 ==这段文字会被高亮== 原生高亮语法，通常渲染为带背景色的文本。
删除线 ~~这段文字被划掉~~ 表示废弃或修改的内容。
嵌套标签 #编程/Rust/工具链 Obsidian 支持层级标签，方便在右侧标签面板中折叠管理。
内部注释 %%这是一段注释%% 只有在编辑模式下可见，阅读模式下完全隐藏。

3. 标注框 (Callouts)

用于醒目地展示警告、提示或引用内容。官方原生支持，极其强大。

基础语法：

Markdown

> [!info] 这是一个提示框标题
> 这里是提示框的正文内容。
> 可以包含多行。

可折叠的 Callout：

Markdown

> [!faq]- 这是一个默认折叠的问答框（注意后面的减号）
> 点开后才能看到这段答案。如果是 `+` 号则是默认展开。

常用类型（替换方括号内的单词即可改变颜色和图标）：

note (笔记), info (信息), todo (待办), warning (警告), danger (危险), bug (Bug), example (示例), quote (引用)。

4. 程序员专属：代码、公式与图表

原生支持所有主流开发语言的高亮、LaTeX 公式以及 Mermaid 图表。

代码块与行内代码：

行内代码使用反引号 `代码`。多行代码块指定语言：

Python

def hello_world():
    print("Keep it simple and DRY.")

数学公式 (MathJax/LaTeX)：

行内公式使用单美元符号包围，例如质能方程 。

独立成行的公式块使用双美元符号：

Mermaid 图表：

直接在代码块语言处写 mermaid，即可渲染流程图、时序图等。

Code snippet

graph TD;
    A[本地修改] --> B{Git 提交};
    B -->|Yes| C[同步到远程库];
    B -->|No| D[继续编辑];

5. 页面元数据 (Properties / YAML)

必须放在整个 Markdown 文件的最顶端（第一行不能有空行），用于定义文档的属性，非常适合配合 Dataview 插件进行自动化数据库查询。

YAML

---
title: "我的第一篇架构笔记"
date: 2026-03-19
tags:
  - 后端
  - 架构
aliases:
  - 架构学习笔记
status: in-progress
---

(在较新版本的 Obsidian 中，这部分会被自动渲染为一个非常直观的 UI 表格进行可视化编辑。)

附录：极简标准 Markdown 备忘（防遗忘）

  • 各级标题： # H1, ## H2, ### H3
  • 粗体与斜体： **粗体**, *斜体*
  • 无序列表： - 列表项
  • 有序列表： 1. 列表项
  • 任务列表： - [ ] 未完成, - [x] 已完成
  • 引用： > 引用文本
  • 超链接： [显示文本](https://...)
