💡 **快速启动提示**：以下所有命令都可通过 `Win + R` 打开“运行”窗口后直接输入调用；也可以在 PowerShell/命令提示符中执行，便于脚本化批量处理。本文按照“核心配置 → 监控诊断 → 硬件管理 → 进阶工具”四个层级，补充每个工具的使用步骤、典型场景和风险提示，帮助在 Windows 上快速完成底层运维与日常排障。
更深入的条目可参考 [system tools plus](system%20tools%20plus.md)。

### 1. 核心配置与底层修改
这些工具直接修改系统行为，适合需要深入自定义、优化或排错的高阶用户。操作前务必做好备份，避免误操作导致系统不可用。
	
#### 注册表编辑器 (Registry Editor)
- **命令**：`regedit`
- **核心价值**：注册表是 Windows 的“配置数据库”，包含开机自启、外设策略、上下文菜单、协议绑定等几乎所有系统开关。遇到界面控件打不开或组策略缺少的项目时，可以直接编辑键值。
- **操作步骤**：
  1. 通过 `Win + R` 输入 `regedit`；启动后按 `Ctrl + L` 直接输入路径，例如 `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run`。
  2. 修改前右键“导出 (Export)”父级键，生成 `.reg` 备份。若需要跨机器部署，可编辑 `.reg` 再批量导入。
  3. 修改完成后使用 `F5` 刷新确认生效；必要时注销用户或重启特定服务以加载新配置。
- **典型场景**：
  - **自定义界面和右键菜单**：在 `*\shell`、`Directory\Background\shell` 等分支中精简冗余项目或加入自研脚本。
  - **修复环境变量界面异常**：卡死或打不开“环境变量”窗口时，直接到 `HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment` 修改 `Path`。
  - **批量部署限制**：锁定 USB 存储、禁用 SMBv1、关闭自动播放等可以统一封装成 `.reg` 下发。
- **风险提示**：某些条目（如 `HKLM\SYSTEM`）与驱动紧密耦合，误删将造成蓝屏。务必逐项核对、避免使用模糊搜索后直接删除。

#### 本地组策略编辑器 (Group Policy Editor)
- **命令**：`gpedit.msc`（专业版、企业版可用）
- **核心价值**：在无需记忆注册表路径的情况下，以树状界面启用/禁用安全与系统策略，内置详细说明和依赖关系。
- **使用技巧**：
  - 善用顶部的“筛选器”和“说明”面板，快速查找关键字，例如“SmartScreen”“Windows Update”。
  - 修改策略后执行 `gpupdate /force` 立即下发，或在“结果集 (Resultant Set of Policy)”中验证是否覆盖到当前用户/计算机。
- **典型场景**：
  - **环境隔离**：在“计算机配置 → 管理模板 → Windows 组件 → Microsoft Defender 防病毒程序”中关闭实时保护、云端提交，确保科学计算或编译缓存不被误删。
  - **更新托管**：在“Windows 更新”节点设为“通知下载并通知安装”，配合“不要在计划时间自动重启”杜绝深夜自动重启。
  - **桌面体验**：关闭锁屏广告、禁用建议、限制商店应用安装。
- **风险提示**：策略最终写回注册表。多台电脑共用一份策略备份时要分清“计算机配置”和“用户配置”，以免覆盖权限。

#### 系统服务管理器 (Services)
- **命令**：`services.msc`
- **核心价值**：全面掌控在后台运行的系统服务和第三方守护进程，可设置启动类型、恢复策略及依赖关系。
- **操作步骤**：
  1. 打开后先按“状态 (Status)”排序，查看正在运行的关键服务。
  2. 双击目标服务，可在“登录”“恢复”“依存关系”标签里查看其运行帐户、故障后自动重启策略以及依赖链。
  3. 修改为“手动 (Manual)”只在调用时启动，“禁用 (Disabled)”则彻底阻止加载。
- **典型场景**：
  - **排查依赖**：发现 WSL、打印、蓝牙等功能不可用时，确认 `LxssManager`、`Print Spooler`、`Bluetooth Support Service` 是否被禁用。
  - **性能优化**：将长期未使用但频繁唤醒硬盘的服务（如 `SysMain`、某些 OEM Telemetry）改为“手动”，释放内存与 I/O。
  - **安全管控**：禁用自启动的第三方常驻服务，防止随系统自动注入。
- **风险提示**：不要禁用名称含 `Remote Procedure Call (RPC)`、`COM`、`DCOM` 等核心组件；若不确定，可先设为“手动”并观察。

### 2. 系统监控与诊断
监控类工具用来找出是谁占用了 CPU、IO、端口或导致系统崩溃。掌握它们可快速定位瓶颈并反馈给运维或开发。

#### 任务管理器 (Task Manager)
- **命令**：`taskmgr` 或 `Ctrl + Shift + Esc`
- **功能概览**：进程管理、性能监控、启动项控制、服务浏览合集。
- **操作技巧**：
  - 在“详细信息”页右键标题栏选择“命令行”“GPU 引擎”等列，可立即看到脚本参数、显卡调度情况。
  - “性能”页可打开资源监视器、“应用历史记录”可统计 UWP 应用使用情况，“启动”页直接禁用开机自动运行程序。
  - “用户”页可为指定会话断开资源占用，适合共享服务器或远程桌面场景。
- **典型问题处理**：
  - **定位内存泄漏**：按“内存”排序，观察进程的“提交大小”变化，配合“分析等待链”判断线程卡在哪个句柄。
  - **隔离测试**：通过“设置关联 (Set affinity)”让 CPU 压力测试或老旧程序只跑在小核/大核，避免拖慢全局。

#### 资源监视器 (Resource Monitor)
- **命令**：`resmon`
- **功能概览**：提供 CPU、磁盘、网络、内存的实时访问明细，列出每个资源的调用者、句柄和 I/O 速率。
- **操作技巧**：
  - “CPU”页可展开“关联句柄”“关联模块”，当文件被锁定时输入关键字直接定位进程。
  - “磁盘”页可通过“总计（B/秒）”找出疯狂读写的文件，双击即可跳转到进程。
  - “网络”页提供端口与远程地址，配合 `netstat -ab` 交叉验证。
- **典型场景**：
  - **端口冲突**：在“网络”页筛选端口，确认是哪个服务占用 3000/8080 等常用端口，结束进程后立即释放。
  - **硬盘异响/卡顿**：查看“最大响应时间”找出拖慢系统的老旧硬盘或异常驱动。

#### 事件查看器 (Event Viewer)
- **命令**：`eventvwr.msc`
- **功能概览**：收集系统、应用、安全等日志，是分析蓝屏、服务崩溃、权限变更的首选。
- **使用技巧**：
  - 在“Windows 日志”中重点关注“应用程序”和“系统”；“自定义视图 → 管理事件”可以把高危事件聚合。
  - 利用“筛选当前日志”，按`事件 ID`（如 1000、7000、4625）或时间范围快速定位。
  - 可将选中的事件保存为 `.evtx` 并附带“显示信息”，方便对接远程支持。
- **典型场景**：
  - **蓝屏/驱动崩溃**：查找“BugCheck”“Kernel-Power (41)”等条目，配合 dump 文件定位驱动。
  - **安全审计**：监控登陆失败 (4625)、特权提升 (4672) 或计划任务变更记录。

### 3. 硬件设备与存储管理
用于初始化磁盘、管理驱动、导出系统拓扑，保障硬件资源被正确识别与分配。

#### 磁盘管理 (Disk Management)
- **命令**：`diskmgmt.msc`
- **核心价值**：图形化分区工具，支持 MBR/GPT 初始化、压缩卷、扩展卷、镜像卷等。
- **操作流程**：
  1. 插入新硬盘后右键“磁盘 x”初始化为 GPT（适用于 2 TB 以上及 UEFI 系统）或 MBR。
  2. 使用“压缩卷”在系统盘挤出空间，再“新建简单卷”安装第二系统或将项目与系统隔离。
  3. 利用“附加 VHD”挂载虚拟硬盘，调试多版本 SQL/AD 测试库。
- **典型场景**：
  - 修复磁盘离线、盘符消失、RAW 分区无法访问等。
  - 为 BitLocker、Storage Spaces 预留未分配空间。

#### 设备管理器 (Device Manager)
- **命令**：`devmgmt.msc`
- **核心价值**：查看硬件设备树、驱动版本、资源冲突；支持更新、回滚、禁用驱动。
- **操作技巧**：
  - 开启“查看 → 显示隐藏的设备”找出残留的虚拟网卡或串口。
  - “属性 → 事件”查看驱动安装历史，必要时回滚到上一个版本。
  - “详细信息”标签选择“硬件 ID”，可直接复制到搜索引擎定位驱动。
- **典型场景**：
  - 解决 USB 调试设备绑定错误、GPU 驱动更新失败、触控板/蓝牙失灵等。
  - 为特定设备禁用电源管理，避免休眠后断连。

#### 系统信息 (System Information)
- **命令**：`msinfo32`
- **核心价值**：导出整机配置清单，包括硬件资源、组件、软件环境，便于远程协助排障。
- **使用方式**：
  - 在“系统摘要”中确认 BIOS 版本、启动模式、固件日期、已安装内存。
  - “硬件资源 → 冲突/共享”可查看 IRQ、DMA 冲突；“组件 → 存储 → 磁盘”列出硬盘固件、接口。
  - “文件 → 导出”生成 `.txt` 报告，附到工单或分享给同伴。
- **典型场景**：核对 TPM/Secure Boot 状态以满足 BitLocker、Windows 11 要求，或验证虚拟化功能是否开启。

### 4. 进阶极客工具箱 (Sysinternals Suite)
Sysinternals 由微软官方维护，可在 <https://learn.microsoft.com/sysinternals/> 下载。无需安装，解压后直接运行，适合深入排查疑难杂症。

#### Process Explorer
- **定位**：进程与 DLL 观察神器，可以实时查看父子进程、线程、句柄与模块调用树。
- **常用技巧**：
  - 使用“查找 → 查找句柄或 DLL (Ctrl + F)”输入文件名/端口，定位被占用的句柄并右键关闭。
  - 开启“选项 → 验证签名”“选项 → VirusTotal.com”判断进程是否安全。
  - 右键进程可直接跳转到“属性”查看 CPU 图、I/O 图、线程堆栈。
- **适用场景**：查杀顽固进程、分析注入模块、确认是谁唤醒 dGPU。

#### Autoruns
- **定位**：全方位启动项管理器，枚举注册表 Run、RunOnce、计划任务、驱动、浏览器扩展、上下文菜单等。
- **使用技巧**：
  - 启动后立即勾选“选项 → 隐藏 Microsoft 项”，聚焦第三方。
  - 通过“保存”导出 `.arn` 快照，与后续快照对比，找出新出现的自启动条目。
  - 右键条目可直接跳转注册表或文件所在路径，方便手工核查。
- **适用场景**：清理顽固启动程序、追踪恶意脚本、部署瘦系统模板。

#### Process Monitor (Procmon)
- **定位**：实时记录文件、注册表、网络、进程/线程操作的“摄像机”，可输出百万级事件。
- **使用技巧**：
  - 启动后立即设置过滤器（例如“进程名 is setup.exe”），否则日志会迅速爆炸。
  - 关注 Result 列的 `NAME NOT FOUND`、`ACCESS DENIED`、`PATH NOT FOUND`，它们通常是崩溃的真正原因。
  - 支持保存 PML 日志，远程共享给同事复现分析。
- **适用场景**：定位安装失败、权限不足、动态链接库加载顺序异常等深层问题。

#### TCPView
- **定位**：网络连接总览面板，实时列出每个进程的本地/远程地址、端口、状态。
- **使用技巧**：
  - 双击条目可跳出详细信息，右键可关闭连接或结束进程。
  - 菜单中勾选“显示未侦听端口”即可观察 UDP 会话。
  - 可导出 CSV 方便进行网络审计。
- **适用场景**：排查内网端口被占、甄别异常回连、验证代理/隧道是否建立。
