# Tor
访问“暗网”（Dark Web）并不像科幻电影里那样复杂，但它**充满了风险**。暗网是指互联网上那些无法被普通搜索引擎（如 Google、Baidu）索引，且必须使用特定软件、配置或授权才能访问的内容。
最常用的访问方式是通过 **Tor 网络**。以下是技术层面的操作步骤，以及**至关重要的安全警告**。
> **⚠️ 严正警告：**
>
> - **高风险环境：** 暗网中充斥着诈骗、恶意软件、非法交易和令人不安的内容。
> - **法律责任：** 虽然浏览暗网本身在大多数国家（包括美国）不违法，但**下载非法内容或参与非法交易**绝对是违法的，执法部门依然有手段追踪。
> - **出于教育目的：** 以下指南仅供技术研究和了解原理，请务必遵守当地法律法规。
------
<a id="step-tor-browser"></a>
### 第一步：获取访问工具（Tor 浏览器）
普通的浏览器（Chrome, Edge, Safari）无法解析暗网的网址。你需要专门的工具。
1. **下载：** 务必只从官方网站下载 **Tor Browser**（Tor 浏览器）。
   - 官网地址：`www.torproject.org`
   - *注意：不要从第三方软件站下载，那些版本可能被植入了后门。*
2. **安装：** 像安装普通软件一样安装它。
3. **连接：** 打开浏览器，点击“Connect”（连接）。Tor 会自动建立加密链路。
   - *如果在国内无法直接连接，你需要配置“网桥 (Bridge)”（见文末）。*
<a id="step-onion"></a>
### 第二步：理解暗网地址（.onion）
暗网的网站地址与普通网站不同：
- **后缀：** 它们以 `.onion` 结尾（普通网站是 .com, .cn）。
- **结构：** 它们通常是一串毫无逻辑的乱码（例如：`vww6ybal4bd7szmgncyruucpgfkqahzddi37ktceo3ah7ngmcopnpyyd.onion`）。
- **特性：** 这些网址无法记忆，且经常变动或失效。
<a id="step-navigation"></a>
### 第三步：如何找到暗网网站（导航）
因为没有 Google，你需要通过“暗网导航”或“暗网搜索引擎”来寻找站点。
在 Tor 浏览器中，你可以尝试输入以下（相对知名的）搜索引擎地址：
1. **The Hidden Wiki (暗网维基):** 这是最著名的导航站之一，列出了许多 `.onion` 链接。
   - *注意：Hidden Wiki 有很多个版本，链接很多是死链或诈骗链接。*
2. **Ahmia:** 这是一个相对“干净”的暗网搜索引擎，它会过滤掉儿童色情等极端恶意内容，适合初学者体验。
   - 可以在普通浏览器搜索 "Ahmia.fi" 找到它的入口，然后在 Tor 中打开其 onion 链接。
3. **Torch:** 一个老牌的暗网搜索引擎。
<a id="step-safety"></a>
### 第四步：最重要的部分——安全生存法则
在暗网“裸奔”极其危险。请务必遵守以下规则：
- **不要最大化窗口：** 保持 Tor 浏览器默认的窗口大小。最大化窗口会让网站获取你的屏幕分辨率，从而形成“浏览器指纹”来追踪你。
- **调整安全级别：** 点击 Tor 浏览器右上角的盾牌图标 -> `Settings` -> `Privacy & Security` -> `Security Level`。建议调整为 **"Safer" (较安全)** 或 **"Safest" (最安全)**。这会禁用 JavaScript，虽然会导致很多网站无法正常显示，但能极大防止恶意脚本攻击。
- **绝对不要输入个人信息：** 不要在暗网上使用你的真实姓名、邮箱、手机号，也不要使用你在明网（普通互联网）上常用的用户名。
- **不要下载任何文件：** PDF、Word 文档等经常被植入木马，一旦下载并打开，你的真实 IP 就可能暴露。
- **遮挡摄像头：** 防止恶意软件通过摄像头偷窥。
- **心理准备：** 你可能会无意中看到极度令人不适的血腥或暴力图片，请做好心理准备，随时准备关闭页面。
------
<a id="summary-tor"></a>
### 总结
访问暗网的操作很简单：**下载 Tor -> 连接 -> 寻找 .onion 链接**。
但在那里，**没有任何法律监管，除了你自己，没人能保护你**。绝大多数普通用户在暗网上除了满足好奇心外，找不到任何有实际价值的内容，反而容易陷入诈骗陷阱（如“低价购买苹果手机”、“比特币翻倍”等全是骗局）。
**如果你在连接 Tor 网络时遇到困难（例如卡在进度条），通常是因为网络封锁。你想了解如何配置“网桥 (Bridge)”来绕过封锁吗？**

# 指纹浏览器
如果让你选一个最**典型**、**行业标杆级**的指纹浏览器来作为“教科书”学习，我会推荐 **AdsPower**。
之所以选它作为例子，是因为它的界面设计非常符合“行业标准”，功能最全，且在跨境电商圈（特别是国内用户）中市场占有率极高。学会了用它，市面上其他的指纹浏览器（如比特浏览器、紫鸟、Hubstudio等）你基本看一眼就会用。
下面为你详细拆解这个“典型代表”长什么样，以及它有哪些核心功能。
------
<a id="fp-core"></a>
### 1. 它的“长相”（界面布局）
打开 AdsPower，你会发现它长得有点像微信电脑版或者一个复杂的邮件客户端。界面通常分为三部分：
- **左侧导航栏：** 核心功能区。包括“账号管理”（最常用）、“应用中心”、“RPA流程”、“费用中心”等。
- **中间主控区：** 这里列出了你创建的所有“浏览器环境”。
  - 每一行代表一个“环境”（也就是一个独立的虚拟电脑）。
  - 你会看到每一行都有：*序号、账号ID、绑定的IP地址、最后打开时间、以及一个绿色的“**打开**”按钮*。
- **顶部工具栏：** 用于批量操作。比如“批量导入账号”、“批量检查IP代理”、“批量更新指纹”。
### 2. 核心功能拆解（它凭什么能防关联？）
作为一个典型的指纹浏览器，AdsPower 把防关联做到了极致，主要体现在以下三个维度的控制：
<a id="fp-core-a"></a>
#### A. 浏览器内核的双重选择
这是它的一个特色。它允许你在创建环境时选择两种内核：
- **SunBrowser（基于 Chrome）：** 适合绝大多数用户，兼容性最好，也就是谷歌浏览器的双胞胎兄弟。
- **FlowerBrowser（基于 Firefox）：** 适合一些对谷歌内核指纹查得特别严的冷门业务，作为替补方案。
> **典型用法：** 你可以把第1个账号设为 Chrome 内核，第2个账号设为 Firefox 内核，让它们看起来完全是两种不同的软件。
<a id="fp-core-b"></a>
#### B. 极其详细的指纹参数（Fingerprint Settings）
当你点开“高级设置”，你会看到它能修改几十项参数，最典型的包括：
- **Canvas（画布）指纹：** 它可以给你的显卡渲染加一点“噪点”，让你的显卡看起来跟别人的不一样，但又很真实。
- **AudioContext（音频）指纹：** 模拟不同的声卡硬件特征。
- **Do Not Track（勿追踪）：** 它可以模拟真实用户的设置习惯。
- **MAC 地址与蓝牙信息：** 直接生成假的硬件地址。
<a id="fp-core-c"></a>
#### C. 应用插件隔离（Extensions）
在普通浏览器里，你装了一个插件，所有网页都能看到。
在 AdsPower 里，你可以给每个环境单独装插件。
- *比如：* 环境A装了“亚马逊卖家助手”，环境B装了“Facebook Pixel helper”，它们互不干扰。这对于避免插件泄露身份非常重要。
### 3. 它的“杀手锏”：RPA 自动化（懒人神器）
这是现代指纹浏览器区别于旧版工具（如虚拟机）的最大亮点。
**RPA（Robotic Process Automation）** 意思是“机器人流程自动化”。AdsPower 内置了一个“积木式”的机器人编辑器。
- **场景：** 假设你有 100 个 Facebook 账号，每天都需要去点赞 3 个帖子，以此来“养号”。
- **传统做法：** 人工登录 100 次，点 300 次赞，累死人。
- **AdsPower 做法：** 你拖拽几个积木块，写好流程——“打开浏览器 -> 访问 Facebook -> 滚动屏幕 -> 随机停留 5 秒 -> 点赞 -> 关闭”。然后点击“批量运行”，这 100 个窗口就会自动弹出来，自己在那操作，你可以去喝咖啡了。
### 4. 费用与门槛
这也是它作为典型代表的原因之一：**丰俭由人**。
- **免费版：** 永久免费提供 **2个** 浏览器环境。这对新手非常友好，你完全可以不花钱先去体验一下流程。
- **付费版：** 按环境数量收费（比如 10 个环境、50 个环境、1000 个环境），还可以按天或按月买。
### 总结：能不能推荐其他替代品？
当然可以。虽然 AdsPower 是典型，但市场上百花齐放，逻辑都一样：
- **追求性价比/免费：** **比特浏览器 (BitBrowser)**。它的界面稍微“极客”一点，但在免费额度和价格上通常比 AdsPower 更激进（更便宜），在群控功能上也很有特色，深受工作室喜爱。
- **追求极致安全/高端：** **Multilogin**。这是指纹浏览器的鼻祖，价格很贵（欧元计费），通常是大企业或对安全要求极高的团队使用。
- **专注电商：** **紫鸟浏览器**。它更像是一个专门为亚马逊卖家定制的“管理软件”，而不纯粹是通用的指纹工具，里面集成了很多电商数据分析功能。
我的建议：
先去下载 AdsPower 的免费版，利用那 2 个免费环境，走一遍“购买代理 -> 配置环境 -> 打开浏览器 -> 访问 IP 查询网站”的流程。弄懂了这个，你就弄懂了整个行业。

# 防封
除了之前提到的“基础操作”，要想彻底洗白一个“脚本号”的嫌疑，你需要从**“深度交互”**和**“资产绑定”**这两个维度入手，让 Google 觉得“封了这个号会误伤真实用户”。
我们可以把账号的安全性看作一个**“信用分”**系统。你现在的分刚及格，以下是具体的加分项（提高真实性）和扣分项（封禁雷区）。
------
<a id="anti-ban-core-1"></a>
### 一、 核心加分项：如何让账号“更有价值”？
Google 的 AI 判断真人的一大逻辑是：**“这个账号是否在互联网上留下了有价值的、连贯的数字足迹？”**
#### 1. “Oauth 登录”大法（最推荐，权重极高）
**原理：** 很多真实用户会用 Google 账号直接登录第三方网站（Sign in with Google）。脚本号通常是一次性的，很少会去绑定这些外部网站。
- **操作：** 去一些国际知名的、高权重的正规网站，选择“使用 Google 账号登录”。
  - *推荐网站：* Pinterest, Quora, Reddit, Medium, Spotify, Zoom, Adobe, 纽约时报等。
  - *效果：* 当你的 Google 账号与这些大平台建立了连接，Google 就会认为这是一个“有人在长期使用”的身份，封号成本会变高。
#### 2. 建立“资产壁垒”（让 Google 不敢轻易删号）
如果一个账号里空空如也，封了就封了。但如果里面有数据，Google 的风控会犹豫。
- **Google Photos（相册）：** 上传几十张生活照片（风景、宠物、食物）。不要上传网图，最好是手机拍的、带有 EXIF 信息（如拍摄时间、设备型号）的照片。
- **Google Drive（云硬盘）：** 上传几个文档，比如 Word 文档、PDF 电子书。
- **Google Maps（地图）：** 这招很管用。在地图上随便收藏几个地点（比如把某个著名的博物馆设为“想去的地方”）。*注意：暂时不要去写评论，写评论容易触发风控。*
#### 3. 模拟“移动端”行为（权重最高的环境）
Google 对来自移动端（手机/平板）的流量信任度远高于 PC 端。
- 如果你之前是在指纹浏览器（模拟 PC）里用的，**不要突然换环境**。
- **进阶操作：** 如果你条件允许，可以在指纹浏览器里把这个环境的 User-Agent 设置为 Android 或 iOS（需要配合相应的分辨率设置），或者直接在一个**干净的手机/云手机**上登录这个号。一旦在手机上稳定登录超过两周，账号极难被封。
#### 4. 产生微小支付（终极护身符）
这是很多老手的“复活甲”。
- **操作：** 如果你有合规的海外信用卡（或正规的虚拟卡），可以在 Google Play 上买一本最便宜的电子书（几美分），或者租一部电影。
- **逻辑：** 骗子和脚本通常不会给 Google 送钱。一旦产生了**付费记录（Billing Profile）**，你的账号性质就从“免费用户”变成了“付费客户”，风控等级会大幅下降。
------
<a id="anti-ban-danger"></a>
### 二、 绝对禁区：封禁雷区与使用规范
既然你的账号已经有过“案底”，以下行为在未来 **3-6 个月内** 都是高压线：
#### 1. 忌“快”与“多”
- **邮件发送：** 每天发送邮件不要超过 5-10 封。绝对不要发带链接的营销邮件。
- **修改资料：** 不要频繁修改头像、名字、生日。改一次之后，半年别动。
- **登录频率：** 不要一会儿登出，一会儿登入。保持**“记住密码”**的常驻登录状态是最好的。
#### 2. 忌“环境剧变”（指纹浏览器的坑）
- **User-Agent 变动：** 很多指纹浏览器升级时，会提示你“更新 Chrome 内核版本”。**千万别点！**
  - *例子：* 你一直用 Chrome 110 版本登录，突然变成 Chrome 120，由于你的 Canvas 指纹和硬件参数没变，但内核变了，这种“老旧硬件跑最新系统”的不协调感会引起怀疑。对于这个老号，**指纹环境一旦配置好，终身不改**。
#### 3. 忌“关联死局”
- **辅助邮箱（Recovery Email）：** 检查你的辅助邮箱是否安全。如果辅助邮箱是你随便填的一个乱码邮箱，或者是一个也快被封的号，赶紧换成一个**你自己常用的、非常稳定的老邮箱**（如 Outlook, Yahoo 或另一个老 Google 号）。
  - *警告：* 如果辅助邮箱被封，这个号很容易被连坐。
#### 4. 忌“IP 漂移”
- 你现在的 IP 可能是静态住宅 IP，这很好。但要小心**WebRTC 泄露**。
- 确保你在指纹浏览器里开启了“禁止 WebRTC”或者“伪装 WebRTC IP”。否则，Google 可能透过代理看到你真实的国内 IP，这种“身在美国，家在中国”的矛盾会秒杀账号。
------
### 总结：一份简单的“养号日程表”
为了帮你度过危险期，建议未来两周按这个节奏来：
| **阶段**       | **动作建议**                                                 | **目的**                   |
| -------------- | ------------------------------------------------------------ | -------------------------- |
| **第 1-3 天**  | 保持挂机，偶尔在 Google 搜索几个关键词（新闻、天气）。**不动任何设置。** | 稳定环境，降低警惕         |
| **第 4-7 天**  | 登录 1-2 个第三方大网站（如 Medium, Quora）；在 YouTube 订阅 1-2 个频道。 | 建立外部链接，模拟真人兴趣 |
| **第 8-14 天** | 往 Google Drive 上传几个非敏感文档；开启两步验证（2FA，如果之前没开）。 | 增加资产，提升安全等级     |
| **第 15 天后** | 可以开始正常的轻度业务操作（如收发少量邮件）。               | 逐步恢复正常使用           |
**记住：** 只要你能像对待一个“真实的人”那样对待这个账号，Google 就没理由封你。所有的封号，都是因为**“你的行为不像人”**。
# 分配机场
<a id="assign-v2rayn"></a>
这是一个非常典型且棘手的问题。
**简单直接的回答是：** 标准版的 Clash（如 Clash for Windows）设计初衷是做一个“总网关”，它通常只通过一个端口（如 7890）把所有流量吸进去，然后在内部根据规则分流，而不是把每个节点都映射成一个独立的端口给你用。
但是，要在指纹浏览器里为不同窗口配置不同节点，你确实需要**“端口 10001 -> 节点 A”、“端口 10002 -> 节点 B”** 这种形式。
针对你“只有机场节点（Clash订阅）”的情况，有三种解决方案，按**推荐程度**排序：
### 方案一：换用 v2rayN（最简单，推荐）
如果你的机场提供 V2Ray/Shadowsocks 订阅链接，建议下载一个 **v2rayN**（Windows上很常用的开源软件）。它原生支持你想要的功能，无需敲代码。
1. **导入订阅**：在 v2rayN 中导入你的机场订阅。
2. **独立映射**：在列表中右键点击你想要用的某个节点 -> 选择 **“设置所选服务器为本地代理”**（或者类似的“分享节点”选项）。
3. **查看端口**：v2rayN 会为此节点开启一个独立的本地端口（比如 10808, 10809 等，可以在设置里自定义）。
4. **填入指纹浏览器**：
   - IP: `127.0.0.1`
   - 端口: `你在 v2rayN 里看到的那个独立端口`
   - 协议: SOCKS5
*优点：图形化操作，想开几个开几个。*
------
### 方案二：使用“Clash多端口转换工具” (进阶)
市面上有一些开发者编写的小工具（通常叫 "Clash2Socks" 或 "Clash-Multi-Port"），专门解决这个问题。
- **原理**：这些工具会读取你的 Clash 配置文件（config.yaml），然后自动在本地开启几十个端口，每个端口对应你订阅里的一个节点。
- **操作**：运行工具 -> 导入订阅 -> 工具会生成一个列表（如：端口 20001 是美国节点，20002 是日本节点）。你直接照着填进指纹浏览器即可。
------
### 方案三：手动修改 Clash 配置文件 (硬核，不推荐)
如果你非要用 Clash for Windows 实现，需要手动修改配置文件 (`.yaml`)，利用 `listeners` 功能（需要 Clash Premium 内核支持）。
1. 打开 Clash for Windows，进入 **Settings** -> **Profile** -> **Parsers** (或者直接编辑当前使用的 `.yaml` 文件)。
2. 你需要添加一段 `listeners` 代码，强行绑定端口。格式极其复杂，示例如下：
   YAML
   ```
   listeners:
     - name: tx-usa  # 自定义名字
       type: socks   # 类型
       port: 10086   # 你想用的本地端口
       proxy: "美国节点A的名字" # 必须和你在Proxy里看到的名字一模一样
   ```
3. **缺点**：机场节点名字经常变，一旦更新订阅，你的配置就失效了，维护起来非常痛苦。
------
### ⚠️ 重要风险提示
既然你使用指纹浏览器，大概率是为了**防关联**（运营电商、社交账号等）。我必须提醒你关于使用“机场节点”的风险：
1. **IP 不纯净**：机场的 IP 是成千上万人共享的。可能你刚登录亚马逊，你的“邻居”刚用同一个 IP 因为违规被封号，你的号也会被连坐（关联封号）。
2. **IP 频繁变动**：机场节点经常掉线或更换入口 IP，这对于账号的安全风控是大忌。
3. **位置跳变**：你今天选的“美国节点1”如果是动态解析的，明天可能就跳到“美国节点2”所在的城市了。
专业建议：
如果是做重要业务（亚马逊、Etsy、Facebook广告等），强烈建议不要用机场/Clash节点。
请去购买专门的 “静态住宅IP (Static Residential IP)”。这种 IP 购买后，服务商会直接给你 IP:端口:账号:密码，你直接填入指纹浏览器即可，既安全又不需要任何客户端中转。

# ip分类
<a id="ip-source"></a>
除了“静态”和“动态”这两个关于**时间**维度的概念外，在购买代理IP时，你最常听到的、也是最关键的分类标准还有以下几个维度。
搞清楚这些名词，能防止你花大价钱买到“假”的高级IP。
### 1. 按“来源”分类（最重要）
这是决定账号是否会被封号的核心指标。
- **机房 IP (Data Center / IDC IP)**
  - **含义**：这些 IP 是阿里云、AWS、Vultr 等云服务器厂商生成的。
  - **特点**：速度极快，价格便宜，数量巨大。
  - **风险**：**极高**。因为普通人不会住在机房里，TikTok 和亚马逊一眼就能认出这是“机器”在访问，而不是“真人”。很多机场节点本质上就是机房 IP。
  - *适用：爬虫抓数据、加速游戏。*
- **住宅 IP (Residential IP / ISP IP)**
  - **含义**：这是真实的美国/英国居民家里宽带的 IP（由 Verizon, AT&T 等运营商提供）。
  - **特点**：速度一般，价格贵，但**信任度极高**。
  - **优势**：平台会认为你就是一个真实的当地居民。
  - *适用：注册账号、养号、电商运营（你的首选）。*
- **移动/手机 IP (Mobile / 4G / 5G IP)**
  - **含义**：来自手机基站的信号 IP。
  - **特点**：**信任度最高**（因为手机 IP 本身就会变，平台对手机 IP 的风控最宽松），但也最贵。
  - **限制**：通常是动态的，很难买到长期固定的手机 IP。
  - *适用：注册高难度账号（如 Instagram, WhatsApp）。*
------
### 2. 按“独享性”分类（决定纯净度）
- **共享 IP (Shared IP)**
  - **含义**：**公交车**。你和几百几千人同时在用同一个 IP。
  - **风险**：这就是“机场节点”的模式。邻居违规，你被连坐。
  - **价格**：非常便宜。
- **独享 IP / 私有 IP (Dedicated / Private IP)**
  - **含义**：**私家车**。在这个时间段内，这个 IP 全球只有你一个人在用。
  - **优势**：非常纯净，只要你不违规，账号就是安全的。
  - **价格**：贵（按个卖，几美元到几十美元一个）。
------
### 3. 动态 IP 里的“行话”
如果你买的是动态住宅 IP（Rotating Residential），你会遇到这两个词：
- **轮换 IP (Rotating)**
  - **机制**：每一次网络请求（刷新一次网页），IP 就自动变一次。
  - *适用：爬虫抓取价格，不适合账号运营（因为账号会瞬间登出）。*
- **粘性 IP / 长效会话 (Sticky Session)**
  - **机制**：虽然是动态池子里的 IP，但你可以设定“我要保持这个 IP **10分钟**或 **30分钟**不准变”。
  - *适用：需要短时间操作一个账号（比如注册、发帖），操作完就扔。*
------
### 总结：你的“防坑”指南
如果你去买 IP，你会看到商家把这些词组合起来卖，价格天差地别。
**请根据你的需求对号入座：**
| **组合名称**          | **含义**                                     | **适合你吗？**                                      |
| --------------------- | -------------------------------------------- | --------------------------------------------------- |
| **共享机房 IP**       | 也就是“机场节点”。多人共用，云服务器来源。   | **仅限看视频**，做运营必死。                        |
| **独享机房 IP**       | 只有你一个人用，但依然是云服务器来源。       | **一般**。容易被平台识别出是代理，不建议做 TikTok。 |
| **动态住宅 IP**       | 真实家庭 IP，但一直在变。                    | **适合跑流量**、注册大量一次性小号。                |
| **静态住宅 IP (ISP)** | **只有你一人用 + 真实家庭宽带 + 永远不变。** | ✅ **这就是你要买的！** 做主账号运营的唯一选择。     |
简单一句话：
你去买代理的时候，认准 “Static Residential (ISP) Dedicated IP”（静态-住宅-独享），这是做指纹浏览器运营的标准答案。
# 两层流量走向
<a id="flow-inbound"></a>
**是的，非常严格。**
这五个词并不是并列关系，而是分属于两个完全不同的**维度**（或者说是两个不同的**工作阶段**）。
你可以把它们理解为**“怎么进站”**和**“往哪坐车”**的关系。你在配置软件时，必须在每一层里各选一个，它们是**同时生效**的。
------
### 第一层：入口层 (Inbound)
**也就是：流量是怎么“进入”到这个软件里的？**
这一层决定了 v2rayN/Clash **“抓取流量的能力”**。
1. **系统代理 (System Proxy)**
   - **方式**：软件把自己设为 Windows 的默认代理。
   - **谁进得来？**：听话的软件（浏览器 Chrome/Edge）。
   - **谁进不来？**：不听话的软件（大部分游戏、命令行、特殊的聊天软件）。
2. **TUN 模式**
   - **方式**：虚拟一张网卡，霸道地接管所有流量。
   - **谁进得来？**：**所有人**。不管你听不听话，只要你想上网，就被强制抓进来。
3. **主要端口监听 (SOCKS/HTTP Port)** —— *这是隐藏的第三种*
   - **方式**：软件只是默默开个门（比如 `10001`），谁都不主动抓。
   - **谁进得来？**：**只有你在设置里手动填了这个端口的软件**（比如你的**指纹浏览器**）。
------
### 第二层：分流层 (Routing)
**也就是：流量进来之后，软件决定把它“送去哪里”？**
这一层决定了流量的**“去向”**。
1. **规则模式 (Rule)**
   - **逻辑**：看地图走。百度的流量扔给本地宽带，Google 的流量扔给机场节点。
2. **全局模式 (Global)**
   - **逻辑**：全部打包。不管是谁，统统扔给机场节点。
3. **直连模式 (Direct)**
   - **逻辑**：全部放行。不管是谁，统统扔给本地宽带（不走代理）。
------
### 它们是如何组合工作的？
这就像是一个**安检站**。
- **第一层（入口）**：你是怎么进安检站的？
  - *System Proxy* = 也就是走普通旅客通道。
  - *TUN* = 把整个火车站封锁了，所有人强制进安检。
  - *Port（指纹浏览器）* = 你有 VIP 邀请函，直接走侧门进来了。
- **第二层（分流）**：你进站后，要把你送上哪列火车？
  - *Rule* = 拿着国内身份证的去坐绿皮车（直连），拿着护照的去坐国际列车（代理）。
  - *Global* = 不管你有啥证件，所有人全部塞进国际列车。
### 针对你的业务（指纹浏览器）的一张图解
你现在的用法其实是非常特殊的，你是**绕过了第一层**，直接利用了第二层（甚至跳过了第二层的判断）。
1. 指纹浏览器 (填入 127.0.0.1:10001)
   ↓
   (跳过 System Proxy，也跳过 TUN)
   ↓
   直接进入 v2rayN 的 10001 号侧门
   ↓
   v2rayN 内部处理
   (因为你指定了端口对应特定节点，所以这里其实连“规则模式”都不看了)
   ↓
   直接发送给“美国节点”
**总结：**
- **TUN / 系统代理**：是给**平时上网**（非指纹浏览器环境）用的，决定了能抓谁的流量。
- **规则 / 全局**：是给**抓进来的流量**做分类用的。
- **指纹浏览器**：它是**特权阶级**，通过端口直连，不受上面这些模式的太多干扰。
