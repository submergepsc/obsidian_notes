在计算机和软件领域，**Portable** 通常翻译为“**便携版**”或“**绿色版**”。
简单来说，它的意思就是：**这个软件不需要安装，点开就能用，而且可以随身带走。**
以下是 Portable 软件的三个核心特点：
### 1. 无需安装 (No Installation)
传统的软件需要运行一个 `setup.exe` 或 `install.msi` 程序，把文件散布到系统文件夹（如 `C:\Program Files`）并修改注册表。而 **Portable 软件通常只是一个文件夹**，你直接双击里面的可执行文件（`.exe`）就可以运行。
### 2. 干净、不留痕迹 (Self-contained)
- **配置随身走：** 软件的所有设置、历史记录和插件都保存在它自己的文件夹里。
- **不污染系统：** 它不会在你的 Windows 注册表或系统目录下留下垃圾文件。当你不需要它时，直接删除那个文件夹即可，不用担心卸载不干净。
### 3. 可移植性 (Portability)
你可以把这个软件文件夹复制到 **U 盘**、移动硬盘或网盘里。
- **场景：** 你在家里配置好了一个浏览器，存了书签和密码。你把这个文件夹考进 U 盘，插到公司或学校的电脑上直接打开，所有的书签和配置都一模一样，就像在用自己的电脑。
------
### 常见后缀与标识
在下载软件时，你经常会看到两个版本供选择：
- **Installer / Setup：** 安装版（需要安装，适合固定在自家电脑使用）。
- **Portable / Zip / 7z：** 便携版（解压即用，适合放在非系统盘或 U 盘）。
### 为什么很多人喜欢 Portable 版？
1. **重装系统不怕丢：** 软件放在非系统盘（如 D 盘），重装系统后直接打开还能用，不需要重新配置。
2. **不占用 C 盘空间：** 既然不需要安装，你就可以把它放在任何地方，避免 C 盘爆满。
3. **多电脑同步：** 配合 Dropbox 或 OneDrive，可以在多台电脑间无缝切换工作环境。
温馨提示：
如果一个软件包含复杂的驱动程序（比如杀毒软件、大型专业设计软件），通常很难做成真正的 Portable 版。这种版本多见于工具类软件（如刚才提到的 WizTree、解压软件、播放器等）。

# 自启问题的解决方法
如果**任务管理器显示“已启用”**，且**启动文件夹里也有快捷方式**，但它依然不启动，这通常是因为 **权限冲突** 或 **Windows 的“快速启动”机制** 导致的。
请按以下顺序排查，通常能解决 90% 的这类顽固问题：
### 1. 最可能的原因：权限问题 (UAC 拦截)
	这是最常见的“隐形杀手”。如果 ManicTime 被设置为“以管理员身份运行”（为了记录某些高权限软件的使用时间），Windows 会**自动拦截**它的普通开机自启，因为开机自启通常不允许自动获取管理员权限。
- **检查方法：**
  1. 找到 ManicTime 的桌面图标或启动文件夹里的快捷方式。
  2. 右键点击 -> **属性 (Properties)** -> **兼容性 (Compatibility)** 选项卡。
  3. **检查：** 看一下“**以管理员身份运行此程序**” (Run this program as an administrator) 是否被勾选了。
- **解决方案：**
  - **方案 A（推荐）：** **取消勾选**“以管理员身份运行”。绝大多数用户不需要这个权限也能正常记录 99% 的软件时间。取消后，重启电脑通常就能正常自启了。
  - **方案 B（必须用管理员权限）：** 如果你必须用管理员权限（否则某些软件记录不到），普通的启动文件夹是无效的。你需要使用 **Windows 任务计划程序 (Task Scheduler)** 来实现“高权限自启”（见文末）。
------
### 2. 系统机制干扰：关闭“快速启动”
Windows 10/11 的“快速启动”本质上是一种休眠，有时候会跳过启动文件夹里的程序加载。
- **解决方案：**
  1. 打开 **控制面板** -> **电源选项**。
  2. 点击左侧的 **“选择电源按钮的功能”**。
  3. 点击顶部的 **“更改当前不可用的设置”**（需要管理员权限）。
  4. 在下方**取消勾选** **“启用快速启动 (推荐)”**。
  5. **保存修改** 并 **重启电脑**（注意是“重启”，不是关机再开）。
------
### 3. “核武器”级解决方案：使用任务计划程序
如果上面两招都不行，或者你必须用管理员权限，这个方法是**百分之百有效**的，它绕过了所有启动文件夹的限制。
1. 在开始菜单搜索并打开 **“任务计划程序” (Task Scheduler)**。
2. 点击右侧的 **“创建任务...” (Create Task)**。
3. **常规 (General) 选项卡：**
   - 名称随便填（如 `ManicTimeAutoStart`）。
   - **关键点：** 勾选底部的 **“使用最高权限运行” (Run with highest privileges)**。
4. **触发器 (Triggers) 选项卡：**
   - 新建 -> 开始任务选择 **“登录时” (At log on)**。
5. **操作 (Actions) 选项卡：**
   - 新建 -> 浏览并选择 ManicTime 的主程序 (`ManicTime.exe`)。
6. **条件 (Conditions) 选项卡：**
   - **取消勾选** “只有在计算机使用交流电源时才启动此任务”（防止笔记本没插电不启动）。
7. 点击确定保存。
总结建议：
先看一眼兼容性设置里有没有勾选“管理员身份运行”，把它关掉通常就立马好了。如果关不掉（业务刚需），就直接用方法 3。
# 自启图标怎么隐藏
想要通过**任务计划程序 (Task Scheduler)** 实现登录启动程序且不显示其启动页面（即“静默启动”或“最小化启动”），通常有几种策略。这取决于该程序本身是否支持特定的命令行参数，或者是否可以通过脚本来控制窗口状态。
以下是实现这一目标的三种常用方法：
------
### 方法一：使用程序的隐藏参数（最简单）
许多程序自带“开机启动且不打开主界面”的参数。
1. 打开 **任务计划程序**，点击右侧的 **创建任务**。
2. 在 **常规** 选项卡，填写名称。
3. 在 **触发器** 选项卡，点击 **新建**，选择 **登录时**。
4. 在 **操作** 选项卡，点击 **新建**，操作为 **启动程序**：
   - **程序或脚本**：浏览选择该程序的 `.exe` 文件。
   - **添加参数 (可选)**：这是关键。尝试输入以下常见参数之一（视程序而定）：
     - `/min`
     - `/hide`
     - `/background`
     - `--hidden`
     - `-silent`
5. 点击确定保存。
------
### 方法二：使用 VBS 脚本实现完全隐藏窗口
如果程序没有自带隐藏参数，你可以通过一个简单的 VBScript 脚本来强制它以隐藏模式运行。
1. 打开记事本，复制并粘贴以下代码：
   VBScript
   ```
   Set WshShell = CreateObject("WScript.Shell")
   ' 将下面的路径替换为你程序的实际安装路径
   WshShell.Run """C:\Program Files\Example\app.exe""", 0, False
   ```
   *注意：路径两侧需要多重引号。`0` 表示隐藏运行。*
2. 将文件另存为 `start_hidden.vbs`（确后缀名是 `.vbs` 而不是 `.txt`）。
3. 在 **任务计划程序** 的 **操作** 选项卡中：
   - **程序或脚本**：输入 `wscript.exe`
   - **添加参数**：输入你保存的脚本路径，例如 `"D:\Scripts\start_hidden.vbs"`。
------
### 方法三：修改任务计划程序的“用户选项”
有时，通过更改任务运行的身份环境，可以防止它弹出交互式界面。
1. 在任务的 **常规** 选项卡中：
   - 找到底部的 **安全选项**。
   - 勾选 **“不管用户是否登录都要运行”**。
2. **注意**：这种方法会让程序运行在后台会话中。这意味着你可能完全看不到它的图标（甚至在系统托盘也看不到），程序只能通过任务管理器关闭。
   > **风险提示**：如果程序需要图形化界面（如托盘图标菜单）进行后续操作，这种方法可能导致你无法与其交互。
------
### 常见问题排查
- **权限问题**：如果程序需要管理员权限才能运行，请在 **常规** 选项卡中勾选 **“使用最高权限运行”**。
- **重复启动**：如果该程序已经在 Windows 设置的“启动”文件夹或注册表里，记得先把那些地方的启动项禁用，否则会打开两个实例。
- **条件限制**：在 **条件** 选项卡中，建议取消勾选 **“只有在计算机使用交流电源时才启动此任务”**，以免笔记本电脑不插电时不启动。

# 多用户管理
在 Windows 系统中，多用户机制设计的核心原则是**“资源共享，环境隔离”**。这意味着虽然两个用户共用一台物理电脑的硬件（CPU、内存、硬盘），但在软件层面（文件、设置、进程）是相互独立的。
以下从**资源管理器（文件系统）**、**应用程序使用**、**系统资源与进程**、以及**注册表与配置**四个维度详细介绍这种情况：
------
### 1. 资源管理器与文件系统 (Resource Manager & Files)
这是用户感知最明显的隔离区域。
- **用户个人文件夹（隔离）：**
  - Windows 为每个用户在 `C:\Users` 下建立了独立的目录（例如 `C:\Users\UserA` 和 `C:\Users\UserB`）。
  - **桌面、文档、下载、图片**等文件夹是完全独立的。
  - **权限控制：** 默认情况下，UserA 无法访问 UserB 的文件夹（会提示无权限）。
  - *特例：* 如果你是**管理员（Administrator）**，你可以通过点击“继续”并进行授权来强制访问另一个标准用户的文件夹。
- **公共文件夹（共享）：**
  - 系统提供了一个 `C:\Users\Public` （公用）目录。放在这里的文档、图片、视频，两个用户都能直接看到和修改。
- **系统盘与软件目录（半共享）：**
  - `C:\Windows` 和 `C:\Program Files` 对两个用户通常是“可见但只读”的。
  - 两个用户都能看到安装了什么软件，但如果没有管理员权限，谁都不能随意删除核心文件。
- **磁盘空间（共享）：**
  - 虽然文件隔离，但硬盘的总容量是共享的。如果 UserA 下载了 500GB 的电影，UserB 会发现硬盘剩余空间变少了，尽管他看不到那些电影文件。
------
### 2. 应用程序使用 (Applications)
应用程序的情况比较复杂，取决于软件是如何安装的。
- **安装范围（All Users vs Current User）：**
  - **为所有用户安装（默认）：** 软件安装在 `C:\Program Files`。UserA 安装后，UserB 的桌面或开始菜单通常也会出现快捷方式，或者 UserB 可以去安装目录直接运行。
  - **仅为当前用户安装：** 某些软件（如 Chrome、VS Code、许多 Python 库）允许安装在 `C:\Users\UserA\AppData` 下。这种情况下，UserA 安装了软件，UserB 根本看不到，也无法运行，除非 UserB 自己也安装一遍。
- **配置与数据（隔离）：**
  - 这是最关键的区别。即使两个用户运行同一个 Chrome 浏览器（安装在 Program Files）：
    - **UserA** 打开全是他的书签、历史记录、保存的密码。
    - **UserB** 打开是一个全新的、空白的浏览器，或者显示 UserB 自己的书签。
  - 这是因为软件的**配置文件**存储在各自的 `AppData` 文件夹（`C:\Users\用户名\AppData`）中，互不干扰。
- **游戏存档：**
  - 现代游戏通常将存档放在“文档”或 AppData 中，所以 UserA 和 UserB 的游戏进度是独立的。
------
### 3. 进程、内存与系统资源 (Processes & Resources)
这里分为两种情况：“注销切换”和“快速用户切换”。
- **情况 A：UserA 注销，UserB 登录**
  - UserA 的所有进程被杀掉，内存清空。UserB 独占硬件资源。
- **情况 B：快速用户切换（同时登录）**
  - UserA 也就是“锁定”状态，UserB 登录进来。
  - **进程隔离：** UserB 打开任务管理器，默认只能看到自己的进程。UserA 的程序（如正在运行的 Word 或下载任务）仍在后台运行，占用内存，但对 UserB 不可见（除非点击“显示所有用户的进程”且拥有管理员权限）。
  - **资源抢占：** 如果 UserA 后台挂着高负载的渲染任务，UserB 前台操作会感觉卡顿，因为 CPU 资源被分走了。
  - **硬件独占冲突：** 某些独占型硬件（如摄像头、声卡特定通道）如果被后台的 UserA 占用，前台的 UserB 可能无法使用。
------
### 4. 注册表与系统设置 (Registry & Settings)
Windows 的注册表是配置的核心，分为“机器层面”和“用户层面”。
- **HKLM (HKEY_LOCAL_MACHINE)：**
  - 存储硬件驱动、系统服务、全局软件设置。
  - 这是**共享**的。UserA 更新了显卡驱动，UserB 登录后用的也是新驱动。UserA 修改了系统时间，UserB 的时间也会变。
- **HKCU (HKEY_CURRENT_USER)：**
  - 存储壁纸、鼠标速度、主题颜色、字体大小、输入法习惯。
  - 这是**独立**的。UserA 可以用深色模式 + 极简壁纸；UserB 可以用浅色模式 + 动漫壁纸。两人互不影响。
------
### 总结对比表
| **功能点**   | **情况描述**   | **备注**                                               |
| ------------ | -------------- | ------------------------------------------------------ |
| **C盘文件**  | 核心隔离       | 只有 `Public` 目录和根目录部分共享                     |
| **软件安装** | 取决于安装选项 | 安装在 Program Files 则共享程序，安装在 AppData 则隔离 |
| **软件设置** | 完全隔离       | 也就是所谓的“千人千面”，每个人配置独立                 |
| **病毒感染** | 可能隔离       | 普通病毒通常只破坏当前用户文件；提权病毒会感染全系统   |
| **回收站**   | 逻辑隔离       | UserA 删的文件，UserB 的回收站里看不到                 |
| **环境变量** | 分层           | 系统变量共享，用户变量隔离                             |
您可以这样理解：
这就好比两个员工（UserA, UserB）共用同一个办公室（电脑硬件）。
- **资源管理器**是他们的办公桌，带锁的抽屉（User 文件夹）只有自己能开。
- **应用程序**是办公室里的复印机，大家都能用（Program Files），但每个人复印出来的文件（AppData/文档）是私有的。
- **注册表**是墙上的空调设置，一部分是针对整个房间的（HKLM），一部分是每个人自己工位的台灯（HKCU）。
------

# 硬盘和硬盘盒的选择
**结论：并不是“一一对应”的，但必须“门当户对”。**
硬盘盒不是专锁某一个硬盘的，它是通用的容器。但是，你必须在**接口物理形状**和**传输协议**这两个核心维度上进行匹配，否则不仅插不进去，插进去也可能烧盘。
针对你软工学生的背景，我把这里的兼容性逻辑拆解为三个层级（由易到难）：
### 1. Level 1：物理尺寸匹配 (能不能装进去)
这是最直观的，分两大类：
- **SATA 接口 (长得像砖头)**
  - **2.5英寸盒**：只能装笔记本硬盘（HDD/SSD）。**注意**：它不需要外接电源，靠 USB 供电。
  - **3.5英寸盒**：能装台式机大硬盘，也能兼容 2.5 英寸小硬盘。**注意**：必须插 **12V 独立电源适配器**，否则带不动大硬盘。
- **M.2 接口 (长得像内存条)**
  - 主流盒子通常支持 **2280 / 2260 / 2242 / 2230** 四种长度（里面有可调节的螺丝柱）。
    - **特例**：最近很火的“2230 专用小盒子”只能装 2230 长度的盘，2280 的长盘插不进去。
### 2. Level 2：协议匹配 (最容易翻车的地方)
这是很多计算机学生都会疏忽的地方。同是 **M.2 接口**，样子长得一模一样，但灵魂（协议）完全不同：
- **NVMe 协议 (M-Key)**：走 PCIe 通道，速度快（读写 1000MB/s+）。
- **SATA 协议 (B+M Key)**：走 SATA 通道，速度慢（读写 ~500MB/s）。
- **坑点**：
  - 如果你买了个 **NVMe 单协议盒子**，插上 **SATA 协议的 M.2 盘**，**不识别**。
  - 反之亦然。
> 🔥 极客推荐 (万能解)：
>
> 去买 “双协议” 硬盘盒。
>
> 认准主控芯片 RTL9210B。这种盒子既支持 NVMe 也支持 SATA，插什么盘都能认，价格只比单协议贵十几块钱（约 60-80 元），兼容性最强。
### 3. Level 3：速率匹配 (别让硬盘受委屈)
硬盘盒的 USB 接口决定了速度上限：
- **5Gbps (USB 3.0)**：适合 SATA 盘，或者你不在乎速度。
- **10Gbps (USB 3.2 Gen2)**：**目前主流**。适合绝大多数 NVMe 盘，跑满约 1000MB/s。
- **40Gbps (USB4 / 雷电)**：极贵（300元+）。除非你用的是三星 990 Pro 这种顶级盘且主要用来剪辑 8K 视频，否则没必要。
------
### 总结对照表 (建议收藏)
| **你的硬盘类型**      | **硬盘接口特征**    | **对应的硬盘盒建议**        | **避坑指南**                    |
| --------------------- | ------------------- | --------------------------- | ------------------------------- |
| **老式笔记本拆机盘**  | 2.5寸 SATA (金手指) | **2.5寸 USB3.0 盒子**       | 几十块钱搞定，别买贵的          |
| **台式机大硬盘**      | 3.5寸 SATA (大砖头) | **3.5寸 带电源盒子**        | **必须带独立供电**，否则带不动  |
| **M.2 (只有1个缺口)** | NVMe (M-Key)        | **NVMe 协议 (10Gbps)**      | 别买成 SATA 协议的盒子          |
| **M.2 (有2个缺口)**   | SATA (B+M Key)      | **SATA 协议** 或 **双协议** | NVMe 盒子不支持它，除非是双协议 |
| **我也分不清**        | M.2 接口            | **双协议盒子 (RTL9210B)**   | **盲选推荐**，通吃所有 M.2      |
一句话建议：
只要你的硬盘是 M.2 的，直接闭眼买“RTL9210B 双协议 10Gbps”硬盘盒，大概率不会翻车。如果是 2.5 寸老硬盘，买个 20 块钱的绿联/奥睿科透明盒子就行。
# 删除只有256g，怎么办
针对手机只有 256GB 存储空间、不支持 SD 卡扩展，但又必须使用大量应用程序的情况，这确实是目前很多用户面临的困境（尤其是 App 体积日益膨胀的今天）。
既然硬件无法改变（不支持 SD 卡），我们需要从**“应用瘦身”**、**“形态替代”**和**“外部扩展”**三个维度来解决。
以下是一套系统的解决方案：
### 1. 驯服“存储巨兽” (微信/QQ)
对于大多数国内用户，微信和 QQ 往往占据了 20GB-100GB 不等的空间。这是最需要清理的地方。
- **微信自带清理（基础）：** `我` -> `设置` -> `通用` -> `存储空间`。清理“缓存”是无害的；清理“聊天记录”需要谨慎。
- **深度管理（进阶）：** 很多时候我们不舍得删聊天记录，但其中的**视频和文件**占了大头。
  - **建议操作：** 定期使用电脑版微信/QQ 进行**备份**（备份到电脑硬盘），然后大胆删除手机上几个月前的媒体文件和聊天记录。
  - **关闭自动下载：** 在微信设置中关闭“自动下载”视频和照片，只看不想存的就别占空间。
### 2. 使用“小程序”和“Web版”替代低频 App
你说“需要使用很多应用”，但其中很多可能只是偶尔用一次（如外卖、购票、甚至某些购物软件）。
- **原则：** 如果一个 App 你一周打开少于 2 次，请卸载它。
- **替代方案：**
  - **微信/支付宝小程序：** 大部分生活类 App（美团、大众点评、滴滴、快递类、点餐类）都有功能完善的小程序。使用它们完全不占额外的安装空间。
  - **PWA / 网页版：** 像知乎、豆瓣、微博等内容消费类平台，可以直接使用浏览器访问，不仅省电还省空间。
### 3. 媒体文件“云端化”或“本地化”
照片和视频通常是第二大存储杀手。
- **云相册（推荐）：**
  - 使用手机品牌的云服务（如小米云、华为云、iCloud等）开启**“优化存储空间”**功能。这样手机里只保留缩略图，原图在云端，需要看时再下载。
  - 或者使用第三方网盘（百度网盘、阿里云盘、Google Photos）开启自动备份，备份成功后**删除本地照片**。
- **本地归档：**
  - 每个月将照片/视频导入电脑或移动硬盘一次，然后清空手机相册。
### 4. 物理外挂：Type-C OTG U盘
既然不支持内置 SD 卡，但现在的手机都支持 **OTG 功能**。
- **方案：** 买一个 **Type-C 接口的手机 U盘**（双口 U 盘，一头插电脑 USB，一头插手机 Type-C）。
- **用法：** 将电影、学习资料、大型文档、甚至很少玩的大型游戏的安装包（apk）存放在 U盘里。需要用的时候插上手机直接读取，或者把拍摄的高清视频直接剪切到 U盘里。这相当于是外接了一张“SD卡”。
### 5. 针对流媒体和缓存的策略
- **在线观看：** 视频 App（B站、爱奇艺等）和音乐 App 非常喜欢缓存内容。请检查它们的设置，将缓存限制设置得低一些，或者养成看完/听完即删的习惯。
- **定期重装：** 很多 App 用久了会产生无法清理的“其他”数据。对于某些体积异常膨胀的 App（除了微信），**卸载并重新安装**通常能瞬间释放几个 GB 的空间。
------
### 总结建议流程
1. **第一步：** 打开手机的“存储空间分析”，看清到底是谁占用了前三名（通常是微信、相册、游戏）。
2. **第二步：** 卸载所有可以用“小程序”替代的 App。
3. **第三步：** 购买一个 128GB 或 256GB 的 **Type-C U盘**（价格通常仅几十元），作为随身仓库。

# 做公众号可能用到
这是一个非常关键的技术问题。很多第一次用 Markdown 写公众号的人都会卡在这里：**Typora 里看着很完美，一复制到公众号后台，图片就全裂了（显示无法加载）。**
这是因为 Typora 里的图片通常是**本地路径**（例如 `C:\Users\hujlo\Pictures\1.jpg`），而微信公众号的服务器无法访问你电脑硬盘里的文件。
鉴于你是 CS 专业的学生，我有两套方案推荐给你：一套是**“现拿现用”的快速方案**（适合这篇只有6张图的文章），另一套是**“一劳永逸”的极客方案**（适合长期写作用）。
------
### 方案一：中间件“渲染”法（最推荐本次使用 🚀）
微信公众号后台原生**不支持** Markdown 语法。如果你直接把 Typora 的源码贴进去，它就是纯文本；如果你把 Typora 预览模式的内容贴进去，格式会乱且图片会挂。
你需要一个**“转换器”**（Markdown 排版工具）。目前圈内最常用的是 **[Markdown Nice (墨滴)](https://www.mdnice.com/)**。
**具体操作流程：**
1. **在 Typora 写文字：** 正常写，图片可以用简单的占位符代替（或者先不管图片，只处理文字）。
2. **复制到 mdnice：** 打开 [mdnice.com](https://www.mdnice.com/)，把 Typora 里的 Markdown 源码复制到左边编辑区。
3. **上传图片（关键步）：**
   - 因为本地图片路径失效，你需要把那 6 张处理好的图片，**直接拖拽**到 mdnice 网页的对应位置。
   - mdnice 会自动把图片上传到它的临时服务器，并生成网络链接。
4. **一键复制到微信：**
   - 点击 mdnice 右上角的**“复制到公众号”**（绿色按钮）。
   - 这个按钮不仅复制内容，还会把 CSS 样式（排版风格）一起打包。
5. **粘贴：** 去微信公众号后台，Ctrl+V。完美！
优点： 不需要配置服务器，操作简单，排版样式非常精美（有很多主题可选）。
缺点： 每次都要手动拖图片。
------
### 方案二：图床 + PicGo 自动化流（CS 学生专属 🤓）
既然你是计算机学生，可能以后会经常写技术博客或文档。我强烈建议你配置 **“Typora + PicGo + 图床”**。
原理：
当你把图片拖入 Typora 时，PicGo（一个开源上传工具）会自动把图片上传到云端（如阿里云 OSS、腾讯云 COS、GitHub、SM.MS 等），然后返还一个 https://... 的链接给 Typora。
**这样你的 Markdown 文件里就全是网络图片，无论粘贴到哪里都能显示。**
**操作流程：**
1. **下载 PicGo：** [PicGo GitHub](https://github.com/Molunerfinn/PicGo)
2. **配置图床：**
   - **免费方案：** 使用 GitHub 做图床（配合 jsDelivr CDN 加速，但有时不稳定）或 SM.MS。
   - **稳定方案（推荐）：** 买一个阿里云 OSS 或腾讯云 COS 的存储桶（非常便宜，一年几块钱）。
3. **关联 Typora：** 在 Typora 设置 -> 图像 -> 上传服务设定中，选择 PicGo。
4. **写作：** 图片拖进去，自动变 URL。
5. **发布：** 依然建议复制到 **mdnice** 进行渲染（为了好看的 CSS 排版），此时不需要再手动拖图片了，直接渲染，然后复制到微信后台。
------
### ⚠️ 一个微信特有的“坑”
即使你使用了方案二（图床），当你把带有 `https://...` 图片的 Markdown 渲染后复制到微信后台时，微信有时会提示**“图片粘贴失败”**或**“外部链接禁止”**（防盗链机制）。
**解决方法：**
- **Markdown Nice 的“解决防盗链”功能：** 它的“复制到公众号”按钮通常会自动处理这个问题（它会把图片转存一下或者把格式调整为微信兼容）。
- **如果还是不行：** 就只能用方案一，手动在 mdnice 里上传图片，这是最稳的。
### 💡 总结建议
对于你现在的**这篇“抢红包”文章**：
**不要折腾 PicGo 了，直接用方案一！**
1. Typora 写好文字。
2. 打开 [mdnice.com](https://www.mdnice.com/)。
3. 粘贴文字。
4. 把你手机传过来的 6 张图，**拖进去**插在对应位置。
5. 选一个像“科技风”或者“极简风”的主题。
6. 点击“复制到公众号”，去微信后台粘贴。
这是效率最高的方法。
