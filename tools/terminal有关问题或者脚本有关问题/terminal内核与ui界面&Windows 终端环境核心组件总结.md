
# **PowerShell** 和 **Windows PowerShell**。

简单来说，它们是**同一个工具的“现代跨平台新版”和“系统自带旧版”的区别**。

以下是两者的核心区别：

### 1. Windows PowerShell (经典自带版)
这是大家最熟悉的、Windows 电脑买来就自带的那个终端。
**路径**：`"C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe"
名称：windows powershell
* **版本：** 通常永远停留在 **5.1** 版本。微软已经宣布停止为其开发重大新功能，只做基础的安全维护。
* **底层架构：** 基于传统的 `.NET Framework`。
* **运行环境：** **仅限 Windows**，和 Windows 系统深度绑定。
* **启动命令：** 系统底层调用的是 `powershell.exe`。
* **优势：** 历史兼容性极强。如果你要在 Windows 上运行非常老旧的系统管理脚本，用它最稳妥。

### 2. PowerShell / pwsh (现代跨平台版)
这是微软后来推倒重做、彻底开源的新版本。为了和旧版区分，它的执行程序改名叫了 `pwsh.exe`，在你的菜单里简称为 `PowerShell`。
**路径**：`C:\Program Files\PowerShell7\pwsh.exe`
**名称**: powershell7
* **版本：** 版本号为 **6.0、7.0 及以上**（目前主流是 7+）。
* **底层架构：** 基于现代的 `.NET Core`（现统称 .NET）。
* **运行环境：** **跨平台**。它不仅能在 Windows 上跑，还能在 macOS 和 Linux 上跑。
* **启动命令：** 系统底层调用的是 `pwsh.exe`。
* **优势：** 性能大幅提升，启动和运行速度更快；加入了大量现代化的编程特性（如并行处理、更智能的错误高亮和自动补全）。它不是系统自带的，通常是用户手动安装，或者安装某些开发工具时被一并装上的。

* **角色**：求稳的“老款发动机”。

* **优势**：极高的向后兼容性，适合运行老旧的企业自动化脚本。

# `wt.exe`与`windowsterminal.exe`区别
### 1. `WindowsTerminal.exe` —— 【被藏起来的“幕后真身”】
* **这是什么**：这就是 Windows Terminal 最核心的、体积最大的本体执行文件。
* **藏在哪里**：因为它通常是通过微软商店（Microsoft Store）安装的，所以它被 Windows 锁死在一个极其隐蔽且权限极严的沙盒目录里。通常在 `C:\Program Files\WindowsApps\` 下面一个名字极长、带有版本号的文件夹里。
* **为什么你平时不用它**：这个目录普通用户默认连**查看权限都没有**。如果你要每次打开 Terminal 都得去这个迷宫一样的深层目录里找 `WindowsTerminal.exe`，那简直是反人类的设计。

### 2. `wt.exe` —— 【摆在明面上的“前台替身 / 任意门”】
* **这是什么**：这是 Windows 专门为 Terminal 创建的一个**“应用执行别名”**（App Execution Alias）。你可以把它理解为一个体积只有 0KB 的超级快捷方式。
* **藏在哪里**：它被放在了你当前用户的 `AppData\Local\Microsoft\WindowsApps` 目录下。最关键的是，这个目录是**默认写进你的系统环境变量 (PATH) 里的**。
* **它的作用**：因为有它的存在，你才可以在电脑的任何地方（运行框、任意文件夹的地址栏、其他代码编辑器里），只要敲下短短的 `wt` 两个字母，系统就会瞬间顺着这个替身，去深宫大院里把真正的 `WindowsTerminal.exe` 唤醒。

---

**💡 极客总结：**

* **`WindowsTerminal.exe` 是本体**，但它在“闭关修炼”，系统不让你轻易碰它。
* **`wt.exe` 是微软发给你的一把“任意门钥匙”**。

你在任务管理器里查户口，或者用全局搜索时，看到的是本体；但你日常在键盘上敲击指挥时，用的永远是 `wt` 这把极简的钥匙。



# 使用管理员权限打开powershell的话会自动调用蓝底的windowersehll
你发现了微软系统里一个极其经典的“祖传顽疾”！

很多玩家刚装完现代版的 `pwsh` 和 `Windows Terminal`，正觉得界面美如画，结果遇到需要系统权限的操作，习惯性地右键点击“以管理员身份运行”，结果**“啪”地一下，那个丑陋的蓝底黑框框又弹出来了**，瞬间打破了沉浸感。

### 为什么会这样？

因为 Windows 系统的底层（特别是开始菜单的右键菜单、Win+X 快捷菜单，以及系统的安全提升机制 UAC）在出厂时，**被微软硬编码（死死绑定）指向了旧版的 `powershell.exe` 和 `conhost.exe`**。

在系统眼里，当你要求“最高权限”时，它为了绝对的安全和稳定，会本能地去调取它最信任的、内置的那个老古董。

---

### 🛠️ 怎么解决？（彻底告别蓝框框）

如果你使用的是 **Windows 11**（或者最新版本的 Windows 10），微软已经提供了一个官方的“终极开关”，可以强行接管所有的命令行启动。

**第一步：将 Windows Terminal 设为“默认终端”**
这是最核心的一步！
1. 打开你的 **Windows Terminal**。
2. 点击顶部的下拉小箭头，进入 **设置 (Settings)**。
3. 在左侧的“启动”菜单里，找到 **默认终端应用程序 (Default terminal application)**。
4. 将它的值从“让 Windows 决定”改为 **Windows Terminal**。
5. 点击右下角的“保存”。

**这一步的作用：** 以后无论系统里有什么软件（包括那些老掉牙的快捷方式）想要呼出 `cmd` 还是老版 `powershell`，都会**被强行劫持**，乖乖地在漂亮的 Windows Terminal 的标签页里打开。

**第二步：如何优雅地开启“管理员模式”的 Terminal？**
设置完第一步后，你以后需要管理员权限时，建议用这两种现代方法：

* **快捷键法（最推荐）**：按下键盘上的 `Win + X`（或者右键点击屏幕左下角的 Windows 徽标），在弹出的菜单中选择 **终端 (管理员)**。这会直接打开一个拥有最高权限的 Windows Terminal（并且会加载你默认配置的新版 `pwsh.6` 内核）。
* **任务栏法**：如果你的任务栏上固定了 Windows Terminal 的图标，你可以按住键盘上的 **`Ctrl + Shift`** 键，然后用鼠标**左键点击**该图标。这在 Windows 里是通用的“以管理员身份运行”的隐藏快捷键。

### 💡 终极极客技巧：使用 `sudo`
既然你习惯用 Linux 的逻辑（熟悉 Python/Rust），你可能会觉得“每次都要新开一个管理员窗口太麻烦了，我能不能在当前的普通窗口里，给某一条命令提权？”

* 如果你的 Windows 11 更新到了最新版（24H2 及以上），微软已经官方引进了 `sudo` 命令。只需在设置 -> 系统 -> 开发者选项中开启 **启用 sudo**。
* 以后在 Terminal 里遇到权限不够的命令，直接像 Linux 一样敲：`sudo winget install xxx`，它就会在当前漂亮的窗口里直接提权，再也不需要去面对那个蓝色的老窗口了！
