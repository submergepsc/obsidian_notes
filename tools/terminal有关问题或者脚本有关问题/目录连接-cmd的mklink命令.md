```shell
cmd:mklink --help


MKLINK [[/D] | [/H] | [/J]] Link Target

        /D      创建目录符号链接。默认为文件
                符号链接。
        /H      创建硬链接而非符号链接。
        /J      创建目录联接。
        Link    指定新的符号链接名称。
        Target  指定新链接引用的路径
                (相对或绝对)。
```

 
### 1. 基础语法结构
你看到的 `MKLINK [[/D] | [/H] | [/J]] Link Target`，其实就是在造句：
* **`Link` (链接)**：你要**新建**的那个“任意门”的名字和路径。
* **`Target` (目标)**：文件或文件夹**真实存在**的路径。

---

### 2. 四种链接模式详解

#### ① 默认模式：文件符号链接 (File Symbolic Link)
* **指令**：`mklink "假文件路径.txt" "真文件路径.txt"` （不加任何字母参数）
* **作用对象**：只能是**单个文件**。
* **大白话解释**：开一个传送门。你修改这个假文件，真文件也会变。但如果你把真文件删了，这个假文件就会变成“死链”（打不开）。
* **注意**：在 Windows 中创建符号链接通常需要**管理员权限**运行 cmd。

#### ② `/D`：目录符号链接 (Directory Symbolic Link)
* **指令**：`mklink /D "假文件夹" "真文件夹"`
* **作用对象**：只能是**文件夹（目录）**。
* **大白话解释**：和上面一样，只不过作用于文件夹。它可以跨磁盘驱动器（比如从 C 盘链到 D 盘），甚至可以链接到局域网里的网络共享文件夹。
* **注意**：同样通常需要**管理员权限**。

#### ③ `/H`：硬链接 (Hard Link)
* **指令**：`mklink /H "假文件.txt" "真文件.txt"`
* **作用对象**：只能是**单个文件**。
* **大白话解释**：**影分身之术**。硬链接不是传送门，而是给同一个物理文件多开了一扇“门”。你把原文件删了，硬链接依然可以正常打开和修改，只有当你把所有硬链接和原文件**全部删光**，这个文件才算真正从硬盘上消失。
* **限制**：硬链接**绝对不能跨盘符**（不能把 C 盘的文件硬链接到 D 盘），因为它操作的是同一个底层的硬盘数据块。

#### ④ `/J`：目录联接 (Directory Junction)
* **指令**：`mklink /J "假文件夹" "真文件夹"`
* **作用对象**：只能是**文件夹（目录）**。
* **大白话解释**：它和 `/D`（目录符号链接）非常像，都可以把一个文件夹映射到另一个地方，且支持跨盘符（C 盘到 D 盘）。
* **为什么首选它？**：因为创建目录联接**不需要管理员权限**，普通用户随时可以创建，而且所有本地软件都能完美识别它，兼容性极好。

# 符号链接和目录联接的区别 

### 1. 权限门槛（最直观的体验差别）
* **目录联接 (`/J`)**：非常亲民。**不需要管理员权限**，任何普通用户随时打开 cmd 就能直接创建。
* **符号链接 (`/D`)**：要求严格。默认情况下，Windows 认为符号链接具有一定的安全风险，因此必须右键选择**“以管理员身份运行”**命令提示符才能创建（除非你的系统开启了“开发者模式”）。

### 2. 网络跨界能力（适用范围）
* **目录联接 (`/J`)**：是个纯粹的“本地人”。它只能在同一台电脑的**本地物理硬盘**之间工作（比如从 C 盘链接到 D 盘，或者外接 U 盘）。它**无法**链接到局域网、NAS 或其他网络共享路径。
* **符号链接 (`/D`)**：是个“外交官”。它不仅能在本地跨盘，还**支持跨网络**。你可以用它把公司局域网共享盘里的文件夹，直接映射成你本地电脑里的一个文件夹。

### 3. “记路”的方式（绝对路径 vs 相对路径）
* **目录联接 (`/J`)**：只认**绝对路径**（死记硬背）。它记住的是目标文件夹的完整门牌号（例如 `C:\Users\Desktop\工作`）。只要这个真文件夹在这个绝对位置，你把联接点剪切到电脑的任何地方，它都能找回去。
* **符号链接 (`/D`)**：更加灵活，支持**相对路径**。它可以被设定为记住“向上一级目录，再向左拐”，这对于程序员打包整个项目代码，或者需要经常整体移动文件夹的场景非常有用 

### 核心区别总结表

| 对比维度 | 目录联接 (Junction `/J`) | 目录符号链接 (Symlink `/D`) |
| :--- | :--- | :--- |
| **创建权限** | ❌ **无需**管理员权限 (随手可建) | ⚠️ **需要**管理员权限 |
| **网络共享/NAS** | ❌ **不支持** (仅限本地硬盘) | ✅ **支持** |
| **相对路径** | ❌ 不支持 (仅支持绝对路径) | ✅ 支持 |
| **兼容性** | 极高 (所有老软件都能完美识别) | 较高 (极少数老旧备份软件可能识别出错) |

**一句话总结：**
对于将本地桌面文件夹接入本地笔记软件这种日常操作，**`/J` 因为无需提权、兼容性极好，是最省心、最完美的选择**。只有当你需要把 NAS、公司共享盘里的文件映射到本地时，才需要麻烦 `/D` 出马。
