# pixpin替代方案
PixPin 在 Windows 上确实是降维打击级别的神器，它的核心灵魂在于三点：**截图标注 + 屏幕贴图 + 离线 OCR**。

在 Ubuntu 生态中，特别是由于现在默认采用 Wayland 显示协议（安全沙盒机制非常严格，不允许软件随便乱截别人的屏幕），目前还没有一个单体软件能 100% 完美复刻 PixPin 的所有功能。

但是，作为习惯了开源和极客工作流的玩家，我们可以通过组合拳完美对标你的需求。下面是 Linux 下最成熟的替代方案：

### 1. 综合截图与标注王者：Flameshot (火焰截图)
这是 Linux 平台绝对的截图一哥，开源且极度轻量。

* **核心优势：** 标注工具异常丰富（箭头、马赛克、序号标记、荧光笔），并且它的环形菜单设计非常极客。
* **关于“贴图”：** 它**支持**把截图直接贴在屏幕上（截图后点击那个“图钉”图标，或者按空格键即可固定）。虽然贴图的管理不如 PixPin 那么精细，但绝对够用。
* **安装命令：**
    ```bash
    sudo apt install flameshot
    ```
* **配置建议：** 像之前配置 CopyQ 一样，进入 Ubuntu 设置把系统的 `PrintScreen` 截屏快捷键接管过来，命令设置为 `flameshot gui`。

### 2. 极致“贴图”体验的原味复刻：Snipaste (Linux 版)
如果你对“贴图”的依赖度极高（比如经常贴很多张代码片段在屏幕上对比），那么直接用 Snipaste 的 Linux 版。

* **核心优势：** 贴图体验和 Windows 版完全一致，支持透明度调节、贴图穿透（鼠标点击可以穿透贴图操作后面的软件）。
* **注意：** 它目前在 Linux 上主要通过 Snap 分发。
* **安装命令：**
    ```bash
    sudo snap install snipaste
    ```

### 3. 补齐 OCR 拼图：NormCap 或 Pot
PixPin 另一个杀手锏是框选直接 OCR 提取文字。Flameshot 和 Snipaste 都不具备这个功能，这就需要专门的工具来补齐。

* **纯粹的 OCR 工具 (NormCap)：** 这是一个完全离线的 Python 开源工具。按下快捷键，屏幕变暗，你框选哪里，哪里的文字就直接进入了你的 `wl-clipboard` 剪贴板。
    * 安装：它推荐使用 pip 安装或者直接下载 AppImage 运行。
* **进阶的翻译/OCR 神器 (Pot)：** 如果你不仅需要 OCR，还需要看文档时划词翻译，强烈推荐开源的 **Pot-app**。它可以监听剪贴板，或者配合快捷键实现极其顺滑的“截图 -> 识别 -> 翻译”一条龙。

**最佳配置路线总结：**
对于日常折腾代码和系统，**Flameshot** 是必装的基础设施，它的标注和基础贴图功能足以覆盖 90% 的场景。建议先把它用 `PrintScreen` 快捷键武装起来，如果后续发现极度缺乏离线 OCR，再补一手 NormCap 即可。




# flameshot使用方法
Flameshot 是 Linux 截图界的“瑞士军刀”，表面上看起来只有一个紫色的图标，但实际上暗藏玄机。尤其是对于经常需要写技术教程或者沉浸在终端工作流里的极客来说，掌握了它的高级特性，效率绝对起飞。

这里为你整理一份 Flameshot 的深度使用指南，直接跳过基础的傻瓜式操作，直击核心玩法：

### 1. 核心交互：打破常规的“转盘菜单”与“右键魔法”

当你通过快捷键（或在终端运行 `flameshot gui`）呼出选区后，你会发现它的工具栏是悬浮在选区边缘的。

* **终极呼出键：按下 `空格键` (Space)**
  这是 Flameshot 最极客的设计。在框选状态下按下空格，你的鼠标指针处会弹出一个**环形的快捷菜单 (Radial Menu)**，你可以直接在里面切换颜色或工具，完全不需要把鼠标移动到屏幕边缘的工具栏上去，手感极其丝滑。
* **右键的妙用：反向操作与马赛克**
  * 在空白处**右键拖拽**：直接呼出**马赛克/模糊 (Pixelate/Blur)** 工具。在写博客发截图时，一秒钟就能把 IP 地址或者 Token 抹掉。
  * 对于任何画出的形状（箭头、框线）：**右键点击**它，可以撤销该形状。
* **滚轮调节：** 选中任何画笔、箭头或马赛克工具时，直接**滚动鼠标滚轮**，可以实时调节线条粗细或模糊程度。

### 2. 写技术博客的神兵利器：自动序号 (Counter)

如果你需要在博客里写类似“第一步配置环境变量，第二步重启服务”的图文教程，Flameshot 的 **自动序号工具 (带有数字 `1` 的气泡图标)** 是绝对的降维打击。

* **使用方法：** 选中该工具后，在截图上每点击一次，就会自动生成一个带圈的递增数字（①, ②, ③...）。
* 配合滚轮调节气泡大小，几秒钟就能在一张复杂的终端输出截图上清晰地标出所有的操作步骤，再也不用去其他软件里手动打字排版了。

### 3. “贴图”功能的正确打开方式 (平替 PixPin)

之前提到过 PixPin 的贴图功能，Flameshot 同样能做到：
* **快捷贴图：** 框选完成后，点击工具栏里的**图钉图标 (Pin)**。
* **更快的盲操：** 框选完成后，如果你不想移动鼠标，直接在键盘上按下与图钉对应的快捷键（通常默认没有绑定单键，你可以在配置里改成某个字母，比如 `P`）。
* **贴图管理：** 贴在屏幕上的图，你可以右键点击它来选择关闭，或者通过鼠标滚轮调节透明度。

### 4. 键盘流快捷键（告别鼠标操作）

在框选状态下，熟练使用这几个按键，能省去大量移动鼠标的时间：

* **`Enter` 或 `Ctrl + C`**：立刻将当前截图（连同标注）复制到咱们刚才配置好的 Wayland 剪贴板里。
* **`Ctrl + S`**：呼出保存对话框。
* **`Ctrl + Z`**：撤销上一步涂鸦。
* **`Esc`**：放弃本次截图。

### 5. 终端联动：写入 `.bashrc` 的自动化魔法

作为命令行玩家，Flameshot 提供了极度丰富的 CLI 参数，你可以把它无缝融进你的终端工作流。

你可以打开你的老朋友 `~/.bashrc`，加入以下别名或函数：

```bash
# 一键全屏截图并直接保存到特定目录（比如你的博客素材库），带时间戳，不弹任何界面
alias shot_full='flameshot full -p ~/Pictures/Screenshots/'

# 延迟 3 秒后呼出截图（用来截取那些一点击就消失的右键菜单或悬浮窗）
alias shot_delay='flameshot gui -d 3000'
```
每次刷新配置后，在终端敲下 `shot_delay`，你就有 3 秒钟的时间去摆好系统界面的“Pose”，然后从容截图。

**一个小建议：** Flameshot 的配置界面（在终端输入 `flameshot config` 呼出）里，有一项“UI 颜色”设置。你可以把它的主色调改成和你终端主题（比如之前折腾的 PowerShell 美化主题或当前的 Ubuntu 终端色系）一致的颜色，让整个系统视觉风格达到完美的统一。
