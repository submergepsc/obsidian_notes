# Bash 入门学习流程

> 适合目标：从“会敲几个命令”到“能理解 Bash 执行逻辑，并写出基础脚本”。
>
> 这份流程基于 `bash知识全景.md` 重新整理，不按知识树死记，而按真正适合初学者的学习顺序来推进。

---

## 0. 学之前先建立一个总认知

先记住一句话：

**Bash 不是一堆命令，而是一套“读取输入 -> 解析 -> 展开 -> 找命令 -> 执行 -> 处理输入输出”的系统。**

你后面遇到的大多数问题，本质上都来自下面几件事：

1. 这行命令到底是谁在执行
2. 变量是在哪里生效的
3. 输出为什么跑到了文件里
4. 引号为什么会改变结果
5. 管道、子 shell、脚本执行为什么会影响变量

所以入门不要先追求“记很多命令”，而是先建立 Bash 的运行模型。

---

## 1. 第一阶段：先会用终端

### 目标

做到这一步时，你应该能：

1. 知道终端、shell、命令三者是什么关系
2. 会运行常见命令
3. 知道标准输入、标准输出、标准错误的大概含义

### 要学什么

1. 终端是什么
2. shell 是什么
3. Bash 是什么
4. 命令是怎么输入并执行的
5. `stdin`、`stdout`、`stderr`

### 建议练习

```bash
pwd
ls
echo hello
date
whoami
```

再感受一下输出和报错的区别：

```bash
ls
ls not_exist_file
```

### 这一阶段先记住

1. 终端是窗口
2. Bash 是解释你输入内容的 shell
3. 命令执行后，正常内容走标准输出，错误信息走标准错误

---

## 2. 第二阶段：理解“命令是怎么找到的”

这一步对应全景里的 **命令解析**，它是 Bash 的核心之一。

### 目标

做到这一步时，你应该能回答：

1. 为什么同一个名字有时不是同一个命令
2. `which`、`type`、`command -v` 为什么结果可能不同
3. alias、function、builtin、外部命令谁优先

### 要学什么

1. `alias`
2. `function`
3. `builtin`
4. `PATH` 查找
5. `type`
6. `command`
7. `hash` 的概念先知道即可

### Bash 找命令的大致顺序

通常可以先这样理解：

1. `alias`
2. shell function
3. shell builtin
4. `PATH` 里的外部可执行文件

### 建议练习

```bash
type echo
type ls
command -v ls
alias ll='ls -alF'
type ll
```

再定义一个函数感受覆盖关系：

```bash
ls() {
  echo "fake ls"
}

type ls
ls
command ls
```

### 这一阶段的关键收获

你要开始意识到：

**你输入的“命令名”，不一定直接对应某个磁盘上的程序。**

---

## 3. 第三阶段：变量系统

这一步是写脚本前的基础。

### 目标

做到这一步时，你应该能理解：

1. Bash 变量和环境变量有什么区别
2. `export` 到底做了什么
3. 当前 shell 和子 shell 里变量为什么不同
4. `$?`、`$1`、`$@` 这些特殊变量是干什么的

### 要学什么

1. 普通变量
2. 环境变量
3. `export`
4. 特殊变量
5. `declare`
6. `readonly`

### 建议练习

```bash
name="tom"
echo "$name"

export city="shanghai"
bash -c 'echo "$city"'

echo $?
```

写一个最小脚本：

```bash
#!/usr/bin/env bash
echo "script name: $0"
echo "arg1: $1"
echo "all args: $@"
```

### 这一阶段的关键收获

1. `a=1` 是 shell 变量
2. `export a=1` 才会把它放进环境，传给子进程
3. 特殊变量是脚本世界里最常用的上下文接口

---

## 4. 第四阶段：展开机制和引号

这部分是初学者最容易乱的地方，但必须早点学。

### 目标

做到这一步时，你应该能避免很多“看起来很玄学”的错误。

### 要学什么

1. 变量展开：`$var`、`${var}`
2. 命令替换：`$(cmd)`
3. 通配符：`*`
4. 双引号和单引号的区别
5. Bash 会先展开再执行这个概念

### 必学规则

1. 双引号 `" "` 保留变量展开，但抑制大多数空格分裂和通配扩展
2. 单引号 `' '` 基本上原样保留内容
3. 变量最好总是写成 `"$var"`
4. 命令替换优先用 `$(...)`，不要优先用反引号

### 建议练习

```bash
name="hello world"
echo $name
echo "$name"
echo '$name'
```

```bash
files=$(ls)
echo "$files"
```

```bash
echo *.md
echo "*.md"
```

### 这一阶段的关键收获

很多 bug 不是命令错了，而是：

1. 展开后内容变了
2. 空格被拆分了
3. 通配符提前展开了
4. 引号没加对

---

## 5. 第五阶段：重定向

这一步会让你真正理解“输入输出流”。

### 目标

做到这一步时，你应该能控制命令的输入输出去向。

### 要学什么

1. `>`
2. `>>`
3. `<`
4. `2>`
5. `2>&1`
6. `/dev/null`
7. 文件描述符 `0`、`1`、`2`

### 最小理解模型

1. `0` 是标准输入
2. `1` 是标准输出
3. `2` 是标准错误

### 建议练习

```bash
echo hello > out.txt
echo world >> out.txt
cat < out.txt
ls not_exist 2> err.txt
ls . not_exist > all.txt 2>&1
ls not_exist > /dev/null 2>&1
```

### 这一阶段的关键收获

1. 屏幕只是默认输出位置
2. 输出和错误可以分开处理
3. Bash 脚本里日志、错误排查、静默执行都离不开重定向

---

## 6. 第六阶段：执行模型

这一步是 Bash 真正的分水岭，也是全景里最重要的部分。

### 目标

做到这一步时，你应该能理解：

1. 为什么管道后的变量经常“没了”
2. 为什么括号里改变量，外面看不到
3. `{}` 和 `()` 的区别
4. 后台执行为什么像是“另一个世界”

### 要学什么

1. 子 shell：`()`
2. 当前 shell：`{}`
3. 管道：`|`
4. 后台：`&`
5. 进程和 shell 的区别
6. `fork / exec` 只要建立直觉，不必一开始死抠

### 核心现象练习

```bash
x=1
(x=2; echo "$x")
echo "$x"
```

```bash
x=1
{ x=2; echo "$x"; }
echo "$x"
```

```bash
echo "hello" | read x
echo "$x"
```

### 你要形成的结论

1. `()` 经常意味着子 shell
2. 子 shell 里的变量修改通常不会回到父 shell
3. 管道常常会引入子进程或子 shell 语义
4. 这就是很多“变量失效”问题的根源

---

## 7. 第七阶段：流程控制

到这里就能开始写有逻辑的脚本了。

### 目标

学会根据条件执行、循环处理、分支判断。

### 要学什么

1. `if`
2. `case`
3. `for`
4. `while`
5. `break`
6. `continue`
7. `test` / `[ ]`

### 建议练习

```bash
if [ -f "/etc/hosts" ]; then
  echo "exists"
else
  echo "missing"
fi
```

```bash
for x in a b c; do
  echo "$x"
done
```

```bash
count=1
while [ "$count" -le 3 ]; do
  echo "$count"
  count=$((count + 1))
done
```

### 这一阶段的关键收获

1. Bash 的条件判断本质上和命令退出状态有关
2. `if` 判断的不是“真/假对象”，而是命令是否成功
3. 循环能让脚本真正具备批处理能力

---

## 8. 第八阶段：函数与脚本

这一步让你从“写一串命令”进入“组织脚本”。

### 目标

做到这一步时，你应该能写一个结构清晰的小脚本。

### 要学什么

1. 函数定义
2. 参数传递
3. `return`
4. `exit`
5. `source`
6. 直接执行脚本

### 建议练习

```bash
hello() {
  echo "hello, $1"
}

hello world
```

理解 `return` 和 `exit`：

```bash
test_fn() {
  return 1
}

test_fn
echo $?
```

### 必须知道

1. `return` 是函数返回状态码
2. `exit` 是整个脚本退出
3. `source a.sh` 是在当前 shell 中执行
4. `./a.sh` 一般是启动一个新的执行环境

---

## 9. 第九阶段：终端与作业控制

这部分能帮你理解交互式使用 Bash 时的行为。

### 要学什么

1. `Ctrl+C`
2. 信号的基本概念
3. `tty`
4. job control
5. `fg`、`bg`

### 建议练习

```bash
sleep 100
```

然后尝试：

1. `Ctrl+C`
2. `Ctrl+Z`
3. `bg`
4. `fg`

### 这一阶段的意义

你会开始理解：

1. shell 不只是执行命令，也在管理前后台任务
2. 终端信号会影响当前前台作业

---

## 10. 第十阶段：进阶安全与工程化习惯

这一阶段不一定要最早学，但一旦开始写正式脚本，就应该补上。

### 要学什么

1. `set -e`
2. `set -u`
3. `set -o pipefail`
4. `trap`
5. `exec`
6. `xargs`

### 推荐最小模板

```bash
#!/usr/bin/env bash
set -euo pipefail

main() {
  echo "start"
}

main "$@"
```

### 这一阶段的关键收获

1. 让脚本更早暴露错误
2. 减少静默失败
3. 让脚本更接近可维护的工程代码

---

## 11. 推荐学习顺序总结

如果你只想知道“到底该按什么顺序学”，就按下面走：

1. 终端、shell、标准输入输出
2. 命令解析
3. 变量系统
4. 展开机制与引号
5. 重定向
6. 执行模型
7. 流程控制
8. 函数与脚本
9. 终端与 job control
10. 高级安全与工程化

这个顺序的核心逻辑是：

**先理解 Bash 怎么运行，再学习 Bash 怎么写脚本。**

---

## 12. 每一阶段的验收标准

### 初级完成

你能做到：

1. 看懂基础命令
2. 知道变量、引号、重定向在干什么
3. 会写简单 `if`、`for`、函数

### 入门合格

你能做到：

1. 理解 alias / function / builtin / PATH 的区别
2. 知道子 shell 和当前 shell 的不同
3. 能解释为什么某些变量在管道后失效
4. 能写出几十行以内的 Bash 脚本

### 继续进阶

你再往后可以学：

1. 数组
2. 参数展开高级写法
3. `trap` 清理逻辑
4. `find` + `xargs`
5. 更规范的脚本错误处理

---

## 13. 最常见的初学者坑

1. 变量两边乱加空格：`a = 1` 是错的，应该是 `a=1`
2. 忘记给变量加双引号，导致空格拆分
3. 把 `if` 当成别的语言里的表达式判断
4. 不理解 `()`、`{}`、`|` 带来的执行环境变化
5. 不区分标准输出和标准错误
6. 不知道 `source` 和直接执行脚本的区别

---

## 14. 一个最小实战路线

建议你按下面 4 个小项目练：

1. 写一个脚本，接收名字参数并打印欢迎语
2. 写一个脚本，检查某个文件是否存在
3. 写一个脚本，遍历当前目录下的 `.md` 文件并输出文件名
4. 写一个脚本，把正常输出和错误输出分别写入不同文件

如果这 4 个都能独立完成，说明你已经不只是“认识 Bash”，而是已经开始真正会用了。

---

## 15. 最后一句学习建议

学 Bash 时，最值得反复问自己的不是：

“这个命令怎么背？”

而是：

1. 这一行在展开后变成了什么
2. 这一行是谁执行的
3. 变量活在哪个 shell 里
4. 输出最后流向了哪里

只要这四个问题越来越清楚，Bash 就会越学越顺。
