# 字体大小调整
原来如此，如果是整个 Ubuntu 系统的字体和 UI 都很小，那说明 Ubuntu 默认的 100% 缩放比例在你的高分辨率屏幕上并不合适。

Linux 下调整全局显示大小，通常有以下三种方案。考虑到你对系统控制力要求较高，我强烈推荐**方案二**，它的显示效果最锐利。

### 方案一：调整全局屏幕缩放（连同 UI、图标一起放大）

这是最基础的系统设置，适合希望将整个系统所有元素等比例放大的情况。

1. 打开 Ubuntu 的 **设置 (Settings)** -> **显示 (Displays)**。
	1. 找到 **缩放 (Scale)** 选项。
2. 尝试将其从 100% 修改为 **125%** 或 **150%**，点击右上角的“应用 (Apply)”。

*注意：如果你的缩放选项里只有 100% 和 200%，你需要先打开面板里的 **“分数缩放 (Fractional Scaling)”** 开关。但在某些系统环境下，开启分数缩放可能会导致部分不支持 Wayland 的应用（比如一些旧版 Electron 应用）出现轻微模糊。*

### 方案二：仅调整字体缩放系数（推荐：更精细、不发虚）

如果你觉得图标大小其实还可以，单纯是文字太小看着费眼，强烈建议使用 GNOME Tweaks 工具**只缩放字体**。这种方式不会触发系统的全局画面重采样，文字边缘会保持绝对的清晰锐利。

1. 打开终端，安装优化工具：
   ```bash
   sudo apt update
   sudo apt install gnome-tweaks
   ```
2. 在应用菜单中搜索并打开 **Tweaks（优化）**。
3. 在左侧边栏选择 **字体 (Fonts)**。
4. 找到底部的 **缩放比例 (Scaling Factor)**。默认值是 `1.00`。
5. 你可以将其微调为 **`1.25`** 或 **`1.35`**，调整时系统的所有字体会实时发生变化，直到你觉得舒适为止。

### 方案三：极客命令行一键修改

如果你不想装图形化界面工具，或者想写个脚本快速配置环境，可以直接用 `gsettings` 修改底层参数。

直接在终端执行以下命令（将 1.25 替换为你想要的倍率）：

```bash
gsettings set org.gnome.desktop.interface text-scaling-factor 1.25
```
*(如果想恢复默认，把数值改回 1.0 即可)*

你可以先试试调整字体缩放系数（方案二/三），这通常能完美解决高分屏下代码编辑器和系统界面的阅读痛点。

### 以下是ubuntu的升级
在 Ubuntu 系统中，“升级”通常分为两种情况：一种是**日常的软件包更新**，另一种是**跨大版本升级**（比如从 22.04 LTS 升级到 24.04 LTS）。

作为习惯掌控命令行的极客，强烈建议你全程使用终端来完成，这样能清楚地看到底层到底在跑什么依赖。

以下是完整的升级指南：

### 情况一：日常软件包和内核更新（推荐定期执行）

这是最常见的更新，只升级当前系统版本内的软件、安全补丁和系统内核。

打开终端，依次运行以下命令：

```bash
# 1. 刷新本地的软件源列表
sudo apt update

# 2. 升级所有已安装的软件包
sudo apt upgrade -y

# 3. 卸载那些因为依赖关系改变而不再需要的孤儿包，保持系统整洁
sudo apt autoremove --purge -y
```

---

### 情况二：跨大版本升级（OS 级别的升迁）

如果你想把整个操作系统升到下一个大版本（比如获取更新版本的 GNOME 桌面、更新的底层 Wayland 支持等），你需要走专门的 Release Upgrade 流程。

#### ⚠️ 升级前的高危防坑警告
大版本升级会重置很多系统级配置，根据你最近对系统的深度定制，升级前务必注意：
* **备份核心资产**：把你本地的个人知识库仓库、代码项目，通过 Git 或你常用的同步工具在其他设备上确认备份完好。
* **自定义配置可能失效**：大版本升级有时会重置显示服务器状态。如果你升级后发现刚刚调好的微信中文输入法又挂了，或者截图工具失效，很可能是系统又默认把你切回了 Wayland，你需要重新检查环境变量或者退回 Xorg。
* **第三方源报错**：升级过程中系统会自动禁用所有的第三方 PPA 源（比如你之前遇到过签名过期的那种源），等升级完你需要手动重新启用它们。

#### 大版本升级步骤：

**1. 确保当前系统已经更新到最新状态**
必须先完成“情况一”里的所有更新，然后再执行：
```bash
sudo apt dist-upgrade -y
```

**2. 检查你的升级策略**
Ubuntu 默认只提示 LTS（长期支持版，两年一次）的升级。检查你的配置文件：
```bash
cat /etc/update-manager/release-upgrades
```
看看最后一行是 `Prompt=lts` 还是 `Prompt=normal`。如果是 `lts`，你只能升级到比如 22.04 -> 24.04。如果是 `normal`，则可以升级到半年一次的非长期支持版（如 23.10）。建议保持 `lts` 以求开发环境的稳定。

**3. 启动大版本升级向导**
在终端执行：
```bash
sudo do-release-upgrade
```
*(如果系统提示找不到新版本，但你确定新版本已经发布，可以加上 `-d` 参数强制查找开发/最新分支：`sudo do-release-upgrade -d`)*

**4. 漫长的等待与交互**
接下来系统会开始计算变更，并弹出一个长篇的英文提示，告诉你将要下载多少数据、删除多少废弃包。
* 按 `y` 确认继续。
* 升级期间，如果终端弹窗询问你**“是否保留你修改过的配置文件，还是替换为包维护者的新版本”**，对于你拿不准的文件，一般选择 `N`（保留你当前的本地修改）。
* 中途**绝对不能**断电或强行关闭终端。

**5. 重启**
跑完所有流程后，终端会提示你重启系统。敲击 `y` 重启，迎接你的就是全新的 Ubuntu 桌面了。

# 使用xorg只有微信能够使用中文输入解决办法
哈哈，看着你用全拼敲出这段话，我已经完全能感受到你对着屏幕的无奈了！别慌，这是切换到 Xorg 之后极其经典的一个“环境变量脱节”问题。

**🔍 为什么会这样？**
在 Wayland 下，输入法和桌面环境是深度绑定的，系统会自动帮你处理好一切。但是切换到老牌的 Xorg 后，你的终端（GTK 架构）和浏览器都需要**明确的环境变量**来告诉它们：“嘿，你要把键盘输入先交给输入法软件处理！” 
微信能输入，是因为微信（无论是 Wine 版还是原生版）内部往往自带了独立的输入法捕捉机制，没按系统的常理出牌。

既然你已经是个熟悉终端和配置文件的极客了，咱们用最标准的 Linux 方式把这个问题彻底解决。

### 第一招：使用系统标准工具 `im-config`（最推荐）

Ubuntu 自带了一个专门用来修复这种输入法环境变量映射的工具。

1. 打开你的终端（用英文敲）。
2. 输入并运行以下命令：
   ```bash
   im-config
   ```
3. 会弹出一个图形界面。点击 **OK**。
4. 接下来会问你是否要显式更新配置，点击 **Yes**。
5. 在列表里，**选择你当前正在使用的输入法框架**：
   * 如果你用的是 Ubuntu 默认的输入法，选择 `ibus`。
   * 如果你是自己折腾安装的搜狗输入法、或者 Fcitx5，选择 `fcitx` 或 `fcitx5`。
6. 点击 **OK**，一路确认到底。

**⚠️ 关键一步：** 配置完成后，**必须注销（Log Out）当前账号并重新登录**（或者直接重启电脑），环境变量才会全局生效。

---

### 第二招：极客手动挡 `~/.xprofile`（如果第一招不奏效）

既然你知道了 `~/.bashrc` 是终端启动时读取的配置，那么在 Xorg 世界里，还有一个对应的文件叫 **`~/.xprofile`**。它是 Xorg 桌面启动时专门读取环境变量的地方。

如果 `im-config` 没能搞定，你可以强行把环境变量写进去。

1. 在终端打开或创建该文件：
   ```bash
   nano ~/.xprofile
   ```
2. 把下面这三行负责“桥接”输入法和软件的核心变量加进去：
   *(注意：下面以 Ubuntu 默认的 `ibus` 为例；如果你用的是 `fcitx`，请把所有的 ibus 替换为 fcitx)*
   ```bash
   export GTK_IM_MODULE=ibus
   export QT_IM_MODULE=ibus
   export XMODIFIERS=@im=ibus
   ```
3. 保存并退出（`Ctrl+O`, 回车, `Ctrl+X`）。
4. 同样的，**注销并重新登录**系统。

重新登录后，你的终端、浏览器就都能完美呼出中文输入法了。赶紧去修复，拯救你的中文输入体验吧！

# 搜索历史命令工具
终端里查找历史命令最经典、最常用的快捷键是：**`Ctrl + R`** (Reverse-i-search)。

### 原生 `Ctrl + R` 的用法：
1. 在终端按下 `Ctrl + R`，你的提示符会变成 `(reverse-i-search)\`': `。
2. 直接输入你记得的**命令片段**（比如 `apt` 或 `wechat`）。
3. 终端会自动显示最近一条匹配的历史命令。
4. **进阶操作**：
   * **翻找上一条**：如果显示的不是你想要的，继续按 `Ctrl + R`，它会继续往更早的历史记录里翻。
   * **执行**：看到想要的命令，直接按 `回车 (Enter)` 就会立刻执行。
   * **编辑**：如果想修改一下再执行，按键盘的 **`右方向键 (→)`**，命令就会落到当前的输入光标处，供你编辑。
   * **退出**：不想找了，按 `Ctrl + G` 或者 `Ctrl + C` 退出查找模式。

---

### 极客进阶推荐：用 `fzf` 接管历史查找

原生的 `Ctrl + R` 只能精确匹配连续的字符串，而且只能看一条，体验比较干瘪。强烈建议你安装现代化的命令行模糊查找神器 **`fzf`**。

它会自动劫持原生的 `Ctrl + R` 快捷键，把它变成一个支持模糊搜索、可以上下预览、全屏交互的超级面板。

**一键安装与配置：**

```bash
git clone --depth 1 https://github.com/junegunn/fzf.git ~/.fzf
~/.fzf/install
```
*(运行 install 脚本时，一路敲 `y` 回车即可，它会自动帮你把快捷键绑定写进配置文件里)*

安装完成后，关闭当前终端重新打开。再次按下 `Ctrl + R`，你就会看到一个极其优雅的交互式历史记录列表，你可以随意输入几个散乱的字母（比如 `sudo qpa`），它就能瞬间帮你把刚才启动微信的那条长命令给揪出来。
**注意**：使用`source ~/.bashrc`来重新启动终端。
